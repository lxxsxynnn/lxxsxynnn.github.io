[CS 스터디] 최단 거리 알고리즘

2026-07-01
  • 최단 거리 알고리즘: 정점(장소) 사이를 잇는 길 중 가장 빠른(가중치가 작은) 길을 찾는 알고리즘
  • 지도 앱에서 “출발지 → 목적지 가장 빠른 길” 찾는 것과 같음
무엇을 구할까? 알고리즘
한 정점에서 나머지 정점까지 다익스트라, 벨만-포드
모든 정점 사이의 최단 거리 플로이드-워셜

1. 다익스트라 알고리즘 (Dijkstra)

쉽게 말하면
출발지에서 가장 가까운 곳부터 하나씩 “여기까지 최단 거리 확정!"
이미 확정된 곳을 거쳐 가면 더 빨라지는 길이 있는지 계속 살펴보며 값을 고쳐 나감
단, 길을 지날수록 시간이 오히려 줄어드는(음수 간선) 경우엔 쓸 수 없음

  • 간선의 가중치가 음수가 아닐 때 사용
  • 방문 안 한 정점 중 가장 비용이 적은 정점을 골라 방문하고, 그 정점을 거쳐 가면 더 싼 길이 생기는지 확인해 값을 갱신
  • 우선순위 큐(힙) 로 “가장 가까운 정점"을 빠르게 꺼내면 효율적

cf) 간선이 음수라는 건
간선의 가중치: 그 길을 지나는 비용(거리·시간·돈 등)
보통은 길을 갈수록 비용이 쌓이니까 양수
음수 간선은 지날 때 비용이 쌓이는 게 아니라 오히려 깎이는(돌려받는) 길

   (A)───5───(B)───(−3)───(C)
   A→B는 5가 들지만, B→C는 오히려 3을 돌려받음
   A→C 총비용 = 5 + (−3) = 2

다익스트라는 “가까운 곳부터 최단 거리 확정 도장“을 찍고 다시 안 돌아봄
그런데 음수 간선이 있으면 확정한 뒤에 더 짧아지는 길이 나타날 수 있어 답이 틀림
그래서 음수 간선이 있으면 벨만-포드를 사용

작동 방식

  1. 시작 정점의 비용은 0, 나머지는 ∞(무한대) 로 둠
  2. 방문 안 한 정점 중 비용이 가장 작은 정점을 방문
    → 그 정점을 거쳐 이웃으로 가는 게 기존보다 더 싸면 이웃의 비용을 갱신
  3. 모든 정점을 방문할 때까지 2를 반복

예시 (시작 정점 A) — 아래 그래프로 한 단계씩 확인

                 (A)
              3 ╱   ╲ 10
               ▼     ▼
             (C) ─4─▶ (B)
            2 │ ╲      │ 2
              │  ╲ 8   │
              ▼   ╲    ▼
             (E)   ╲─▶ (D)
              │         ▲
              └─── 5 ───┘

   간선: A→B(10) A→C(3) C→B(4) C→E(2) C→D(8) B→D(2) E→D(5)

규칙: 방문 안 한 정점 중 거리가 가장 작은 정점을 확정하고
그 정점을 거쳐 이웃으로 가는 게 더 비용이 적으면 이웃 거리를 갱신(∞는 아직 도달 못 한 상태)

시작 — A만 0, 나머지는 ∞

방문 A B C D E
– 0 ∞ ∞ ∞ ∞

① A 방문 (거리 0 확정) — A에서 나가는 길 갱신

  • A→B : 0 + 10 = 10 → B = 10
  • A→C : 0 + 3 = 3 → C = 3

(A에서 갈 수 있는 B·C만 값이 생김)

방문 A B C D E
A 0 10 3 ∞ ∞

② C 방문 (남은 것 중 최소 거리 3) — C를 거쳐 가는 길 갱신

  • C→B : 3 + 4 = 7 < 10 → B = 7 ✅ 갱신
  • C→E : 3 + 2 = 5 < ∞ → E = 5
  • C→D : 3 + 8 = 11 < ∞ → D = 11
방문 A B C D E
C 0 7 3 11 5

③ E 방문 (남은 것 중 최소 거리 5) — E를 거쳐 가는 길 갱신

  • E→D : 5 + 5 = 10 < 11 → D = 10 ✅ 갱신
방문 A B C D E
E 0 7 3 10 5

④ B 방문 (남은 것 중 최소 거리 7) — B를 거쳐 가는 길 갱신

  • B→D : 7 + 2 = 9 < 10 → D = 9 ✅ 갱신
방문 A B C D E
B 0 7 3 9 5

⑤ D 방문 (남은 것 중 최소 거리 9) — 더 갱신할 이웃 없음 → 종료

방문 A B C D E
D 0 7 3 9 5

최종 최단 거리: A=0, C=3, E=5, B=7, D=9
D가 11 → 10 → 9로 더 짧은 길이 발견될 때마다 값이 줄어드는 게 핵심

2. 벨만-포드 알고리즘 (Bellman-Ford)

쉽게 말하면
모든 길을 여러 번 반복해서 “이 길로 가면 더 빨라지지 않아?” 하고 계속 확인하는 방식
다익스트라보다 느리지만, 음수 간선이 있어도 잘 동작
만약 아무리 반복해도 계속 값이 줄어든다면 → 돌수록 짧아지는 이상한 고리(음의 사이클) 가 있다는 뜻

  • 특정 정점에서 다른 정점까지 최단 거리를 구함 (간선 가중치가 음수여도 가능)
  • 음의 사이클(돌수록 비용이 무한히 줄어드는 고리)이 있으면 최단 경로가 정의되지 않아 사용 불가

왜 간선을 n-1번만 순회할까? (정점이 n개일 때)

  • 한 정점에서 다른 정점까지의 최단 경로는 최대 n-1개의 간선으로 만들어짐
    (정점이 4개면 아무리 멀리 돌아도 간선 3개면 다 이어짐)
  • 그래서 전체 간선을 n-1번 반복하면 모든 최단 거리가 확정됨

예시 (다익스트라와 같은 그래프, 시작 정점 A) — 결과는 같지만 과정이 다름

  • 정점 5개 → n-1 = 4회전 반복
  • 간선 검사 순서: A→B, B→D, C→B, C→D, C→E, E→D, A→C (순서는 임의로 고정)
  • 각 회전마다 모든 간선을 훑으며 출발점 거리 + 간선 값 < 도착점 거리이면 갱신
   ┌── 시작 ── A=0, 나머지 ∞
   │
   1회전 → 2회전 → 3회전 → 4회전
   회전마다 모든 간선을 한 번씩 검사

시작: A만 0, 나머지 ∞

회전 A B C D E
– 0 ∞ ∞ ∞ ∞

1회전 — 모든 간선 검사 (갱신된 것만 표기)

  • A→B : 0 + 10 = 10 → B = 10
  • B→D : 10 + 2 = 12 → D = 12
  • A→C : 0 + 3 = 3 → C = 3
회전 A B C D E
1 0 10 3 12 ∞

2회전 — C·E까지 값이 생겨 더 짧은 길이 보임

  • C→B : 3 + 4 = 7 < 10 → B = 7 ✅
  • C→D : 3 + 8 = 11 < 12 → D = 11 ✅
  • C→E : 3 + 2 = 5 → E = 5
  • E→D : 5 + 5 = 10 < 11 → D = 10 ✅
회전 A B C D E
2 0 7 3 10 5

3회전 — B가 7로 줄어든 덕에 D가 한 번 더 줄어듦

  • B→D : 7 + 2 = 9 < 10 → D = 9 ✅
회전 A B C D E
3 0 7 3 9 5

4회전 — 모든 간선을 검사해도 갱신 없음 → 최단 거리 확정

회전 A B C D E
4 0 7 3 9 5

간선 검사 순서가 나빠서 값이 한 칸씩 늦게 퍼짐 (D: 12 → 11 → 10 → 9)
그래서 순서와 무관하게 정답을 보장하려면 n-1회전이 필요
결과는 다익스트라와 동일(A=0, B=7, C=3, D=9, E=5)하지만
다익스트라는 정점을 한 번씩만 확정하고 벨만-포드는 전체 간선을 여러 번 반복

  • 음의 사이클 감지: n-1회전을 다 돌린 뒤 한 번 더(5회전) 돌렸는데도 값이 또 줄어든다면?
    → n개 이상의 간선으로 최단 거리가 갱신됐다는 뜻이라 모순 → 음의 사이클이 존재한다는 신호

다익스트라 vs 벨만-포드

구분 다익스트라 벨만-포드
음수 간선 불가 가능
음의 사이클 감지 못 함 감지 가능
방식 가까운 정점부터 확정 모든 간선을 n-1번 반복
시간 복잡도 O(E log V) (우선순위 큐) O(VE)
속도 빠름 상대적으로 느림

음수 간선이 없으면 빠른 다익스트라, 음수 간선이 섞여 있으면 벨만-포드를 사용
