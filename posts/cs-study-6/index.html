[CS 스터디] 캐시 메모리

2026-06-05

1. 캐시 메모리와 지역성

캐시 메모리 (Cache Memory): CPU의 처리 속도와 메인 메모리(RAM)의 접근 속도 차이를 줄이기 위해 사용하는 고속 저장소

  • 원리: 자주 사용하는 데이터를 속도가 빠른 캐시에 미리 복사해 두어, 매번 메인 메모리까지 가지 않고도 데이터를 빠르게 읽어올 수 있게 함
  • 구조: 주로 L1, L2, L3 캐시로 나뉘며 숫자가 작을수록 CPU와 가깝고 속도가 빠름

참조 지역성 (Locality of Reference)

캐시는 한정된 공간이므로, 어떤 데이터를 저장할지가 매우 중요함. 이때 기준이 되는 것이 지역성

  1. 시간 지역성 (Temporal Locality): 최근에 참조된 데이터는 가까운 미래에 다시 참조될 가능성이 높다는 원리
    • 예: 반복문의 인덱스 변수(i), 누적 합계 변수 등
  2. 공간 지역성 (Spatial Locality): 참조된 데이터와 인접한 주소의 데이터가 곧 참조될 가능성이 높다는 원리
    • 예: 배열(Array) 데이터의 순차적 접근

2. 캐시 히트와 캐시 미스

  • 캐시 히트 (Cache Hit): CPU가 요청한 데이터가 캐시에 있는 경우. 즉시 반환하여 성능이 극대화됨
  • 캐시 미스 (Cache Miss): 요청한 데이터가 캐시에 없어 메인 메모리에서 가져와야 하는 경우. 지연 시간(Latency)이 발생함
  • 적중률 (Hit Ratio): (캐시 히트 횟수) / (전체 참조 횟수). 일반적으로 90% 이상의 적중률을 가짐

3. 캐시 메모리의 매핑 방식

메인 메모리는 캐시보다 훨씬 크기 때문에, 수많은 메모리 블록이 좁은 캐시 공간을 나누어 써야 함
이때 어떤 규칙으로 데이터를 배치할지 결정하는 것이 매핑 방식

1) 직접 매핑 (Direct Mapping)

  • 비유: “지정된 우편함” (101호, 201호, 301호는 무조건 1번 우편함만 사용)
  • 원리: 메모리 주소에 따라 캐시 내의 저장 위치가 미리 딱 정해져 있는 방식
  • 특징: 위치가 정해져 있어 찾기 매우 빠르지만,
    같은 칸을 사용하는 다른 주소의 데이터가 들어오면 기존 데이터를 계속 밀어내야 하는 **충돌(Collision)**이 잦음

2) 연관 매핑 (Associative Mapping)

  • 비유: “빈 방 아무 데나 들어가는 호텔”
  • 원리: 정해진 위치 없이 캐시의 빈 공간 어디든 데이터를 저장할 수 있는 방식
  • 특징: 빈 공간을 알뜰하게 써서 충돌은 적지만, 나중에 특정 데이터를 찾을 때 캐시 전체를 다 뒤져봐야 하므로 검색 속도가 느림

3) 세트 연관 매핑 (Set-Associative Mapping)

  • 비유: “구역이 정해진 전용 주차장”
  • 원리: 캐시를 여러 개의 **세트(Set)**로 나누고, 특정 주소는 지정된 세트 내의 아무 칸에나 들어갈 수 있게 한 방식
  • 특징: 직접 매핑의 빠른 속도와 연관 매핑의 높은 효율을 절충한 방식으로, 현대 CPU에서 가장 널리 사용됨
    (예: “A구역(세트)에 주차하되, 그 안에서는 빈 칸 아무 데나 쓰세요”)

4. 매핑 방식 요약 비교

구분 직접 매핑 연관 매핑 세트 연관 매핑
매핑 위치 지정된 하나의 라인 모든 라인 가능 지정된 세트 내 모든 라인
장점 구현 단순, 속도 빠름 적중률 매우 높음 적중률과 속도의 균형
