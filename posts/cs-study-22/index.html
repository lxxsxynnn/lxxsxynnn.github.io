[CS 스터디] 정렬 알고리즘 1

2026-06-26

1. 버블 정렬 (Bubble Sort)

  • 이웃한 두 값을 비교하며 크기 순으로 정렬하는 방식
  • 한 번 순회할 때마다 가장 큰 값이 배열의 뒤로 밀려나며 정렬됨
  • 연속된 두 값을 차례대로 a(왼쪽), b(오른쪽)라 할 때
    • b ≥ a 이면 이미 정렬된 상태로 판단
    • b < a 이면 두 값의 위치를 교환
  • 이 과정을 정렬이 완료될 때까지 반복

정렬 과정 (배열 [5, 1, 4, 2])

1회전: 가장 큰 값(5)을 맨 뒤로
   [5, 1, 4, 2] → 5>1 교환 → [1, 5, 4, 2]
   [1, 5, 4, 2] → 5>4 교환 → [1, 4, 5, 2]
   [1, 4, 5, 2] → 5>2 교환 → [1, 4, 2, |5]   ← 5 확정

2회전:
   [1, 4, 2] → 1<4 유지 → [1, 4, 2]
   [1, 4, 2] → 4>2 교환 → [1, 2, |4, 5]      ← 4 확정

3회전:
   [1, 2] → 1<2 유지 → [1, |2, 4, 5]          ← 정렬 완료
  • n번째 요소를 정렬하는 데 n-1번 비교 → 전체적으로 n(n-1)/2번 연산
  • 시간 복잡도 O(n²) — 비교적 느리지만 별도의 메모리 공간이 필요 없음

2. 선택 정렬 (Selection Sort)

  • 배열을 순회하며 앞에서부터 각 자리에 들어갈 값을 선택해 위치시키는 방식
  • 첫 번째 자리에는 최솟값, 두 번째 자리에는 그다음으로 작은 값을 차례로 배치
  • 인덱스 i에 넣을 값을 고를 때 0 ~ i-1은 이미 정렬이 끝난 상태
  • 남은 구간에서 찾은 최솟값을 인덱스 i의 값과 교환하며 마지막까지 반복

정렬 과정 (배열 [5, 1, 4, 2])

   [5, 1, 4, 2]   남은 구간 최솟값 1 ↔ 5 교환 → [1, | 5, 4, 2]
   [1, 5, 4, 2]   남은 구간 최솟값 2 ↔ 5 교환 → [1, 2, | 4, 5]
   [1, 2, 4, 5]   남은 구간 최솟값 4 (제자리)  → [1, 2, 4, | 5]
   [1, 2, 4, 5]   정렬 완료
  • 인덱스 i의 값을 찾기 위해 n-i개를 비교 → (n-1) + (n-2) + … + 1 = n(n-1)/2
  • 시간 복잡도 O(n²) — 느린 편이지만 추가 메모리가 필요 없고 구현이 간단함

3. 삽입 정렬 (Insertion Sort)

  • 배열을 앞에서부터 순회하며 정렬된 부분의 알맞은 위치에 값을 삽입하는 방식
  • 인덱스 i의 값 a를 정렬할 차례일 때 0 ~ i-1은 이미 정렬된 상태
  • 정렬된 구간에서 a보다 작거나 같은 수와 큰 수 사이에 a를 삽입

정렬 과정 (배열 [5, 1, 4, 2])

   [5 | 1, 4, 2]   1을 [5] 앞에 삽입       → [1, 5 | 4, 2]
   [1, 5 | 4, 2]   4를 1과 5 사이에 삽입    → [1, 4, 5 | 2]
   [1, 4, 5 | 2]   2를 1과 4 사이에 삽입    → [1, 2, 4, 5]
   정렬 완료
  • 각 순회에서 인덱스 i의 값을 알맞은 위치에 넣기 위해 최대 n-i번 탐색
  • 1 + 2 + … + (n-1) = n(n-1)/2번 연산 → 시간 복잡도 O(n²)
  • 거의 정렬된 배열에서는 비교가 적게 일어나 최선의 경우 O(n) 으로 빠름

4. 합병 정렬 (Merge Sort)

  • 재귀를 이용하는 분할 정복(Divide and Conquer) 알고리즘
    • 분할: 배열을 절반씩 쪼갬
    • 정복: 분할한 배열을 정렬하면서 하나로 합병
  • 크기가 0 또는 1이 될 때까지 절반씩 분할한 뒤, 다시 합치며 정렬 수행

정렬 과정 (배열 [5, 1, 4, 2])

   분할                         합병(정렬하며 합침)
   [5, 1, 4, 2]                    [1, 2, 4, 5]
       ├── [5, 1]      ──▶         [1, 5]
       │     ├── [5]                 ▲
       │     └── [1]                 │
       └── [4, 2]      ──▶         [2, 4]
             ├── [4]                 ▲
             └── [2]                 │
  • 합병할 때는 두 배열과, 둘을 합칠 크기의 빈 배열이 필요함
  • 두 배열의 앞 요소부터 비교해 작은 값을 차례로 빈 배열에 채워 정렬

cf) 분할 정복: 해결하기 어려운 문제를 작은 문제로 나눠 해결하는 방식

  • 분할에 O(log n), 각 단계의 합병에 O(n) → 시간 복잡도 O(n log n)
  • 빠르지만 합병용 추가 배열이 필요해 별도의 메모리 공간을 사용함

정렬 알고리즘 비교

알고리즘 평균 시간 복잡도 최악 시간 복잡도 추가 메모리 특징
버블 정렬 O(n²) O(n²) O(1) 구현이 단순하나 느림
선택 정렬 O(n²) O(n²) O(1) 교환 횟수가 적음
삽입 정렬 O(n²) O(n²) O(1) 거의 정렬된 데이터에 강함
합병 정렬 O(n log n) O(n log n) O(n) 안정적이나 추가 공간이 필요함
