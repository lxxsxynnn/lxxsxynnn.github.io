[CS 스터디] 최소 신장 트리

2026-06-30
  • 신장 트리(Spanning Tree): 그래프의 모든 정점을 포함하는 트리 (사이클이 없음)
  • 최소 신장 트리(MST, Minimum Spanning Tree): 가중치 그래프에서 간선의 가중치 총합이 가장 작은 신장 트리

아래 두 알고리즘은 동일한 예시 그래프를 사용합니다. (정점 4개, MST 가중치 합 = 7)

         1
    (A)─────(B)
     │      / │
    3│   2/   │4
     │  /     │
    (C)─────(D)
         5

1. 프림 알고리즘 (Prim’s Algorithm)

  • 임의의 정점을 시작점으로 트리를 확장하며 MST를 만드는 방식
  • 현재 트리에 연결된 간선 중 가중치가 가장 작은 간선의 정점을 선택하는 과정을 반복
  • 모든 정점이 포함될 때까지 트리를 확장

cf) 그리디 알고리즘 각 단계의 최선의 선택이 모여 전역 최적해를 찾는 방식
앞의 선택이 이후 선택에 영향을 주지 않아야 하고, 지역적 최선이 전역적 최적이 되어야 함

정렬 과정 (정점 A에서 시작)

단계 트리에 포함된 정점 선택 가능한 간선 선택
① {A} A-B(1), A-C(3) A-B (1)
② {A, B} A-C(3), B-C(2), B-D(4) B-C (2)
③ {A, B, C} B-D(4), C-D(5) B-D (4)
④ {A, B, C, D} 모든 정점 포함 → 종료 — 
   ①        ②          ③
   (A)      (A)─(B)     (A)─(B)
            1           1   │4
                   2        │
              (C)         (C)   (D)
   → 선택 간선: A-B(1) + B-C(2) + B-D(4) = 7

2. 크루스칼 알고리즘 (Kruskal’s Algorithm)

  • 간선을 기준으로 트리를 만드는 그리디 알고리즘
  • 간선을 가중치 오름차순으로 정렬한 뒤, 작은 간선부터 차례로 선택
  • 단, 간선을 선택했을 때 사이클이 생기면 제외하고 다음 간선을 확인
  • 모든 정점이 연결되면 종료
  • 흩어진 여러 트리를 작은 간선으로 연결해 하나의 MST로 합침

정렬 과정 (간선 오름차순: A-B(1), B-C(2), A-C(3), B-D(4), C-D(5))

단계 확인 간선 사이클 여부 선택 여부
① A-B (1) 없음 선택
② B-C (2) 없음 선택
③ A-C (3) A·C 이미 연결됨 제외
④ B-D (4) 없음 → 전부 연결 선택 
   ①        ②          ④
   A─B      A─B         A─B
   1        1 │2        1 │2 │4
              C           C  D
   → 선택 간선: A-B(1) + B-C(2) + B-D(4) = 7

cf) 유니온 파인드(Union-Find) 알고리즘 두 원소가 같은 집합에 속하는지 판단하는 알고리즘. 그래프에 루트가 없으므로 대표 노드를 정해 집합을 구분한다.

  • union: 두 노드를 하나의 집합으로 합치는 연산
  • find: 특정 노드가 속한 집합의 대표 노드를 찾는 연산

크루스칼에서는 사이클 방지를 위해 사용 — 두 정점이 다른 집합이면 연결(union)하고, 같은 집합이면 사이클이 생기므로 연결하지 않는다.

프림 vs 크루스칼

구분 프림 알고리즘 크루스칼 알고리즘
기준 정점 중심 트리 확장 간선 중심 트리 병합
핵심 자료 우선순위 큐(힙) 간선 정렬 + 유니온 파인드
시간 복잡도 O(E log V) O(E log E)
유리한 경우 간선이 많은 밀집 그래프 간선이 적은 희소 그래프
