[CS 스터디] 스케줄링

2026-06-03

1. 스케줄링의 목적

운영체제는 한정된 CPU 자원을 효율적으로 관리하기 위해 여러 프로세스 사이에서 실행 순서를 결정함

  • 공평성: 모든 프로세스가 공평하게 자원을 배분받아야 함
  • 효율성: 시스템 자원(CPU 등)이 쉬지 않고 최대한 활용되도록 관리
  • 안정성: 우선순위가 높은 프로세스(커널 프로세스 등)를 먼저 처리
  • 반응 시간 보장: 사용자의 요청에 대해 적절한 시간 내에 응답
  • 무한 연기 방지: 특정 프로세스의 작업이 무기한 연기되는 기아(Starvation) 현상을 방지

2. 스케줄링의 단계

스케줄링은 범위와 목적에 따라 크게 세 가지 단계로 구분됨

장기 스케줄러 (Long-term Scheduler)

  • 역할: 준비 큐에 어떤 프로세스를 넣을지 결정하여 메모리에 올라가는 프로세스의 수(Degree of Multiprogramming) 를 조절
  • 특징: 잡(Job) 스케줄러라고도 하며, 현대의 시분할 시스템에서는 대부분 사용하지 않고 프로세스 생성 시 바로 준비 상태로 진입함

중기 스케줄러 (Mid-term Scheduler)

  • 역할: 메모리에 로드된 프로세스 수를 동적으로 조절하여 시스템 부하를 관리
  • 스와핑(Swapping): 메모리가 부족할 때 일부 프로세스를 통째로 저장 장치로 내보내고(Swap-out), 필요할 때 다시 불러옴(Swap-in)
  • 상태 변화: 스왑 아웃된 프로세스는 ‘중단(Suspended)’ 상태가 되어 실행이 일시 중지됨

단기 스케줄러 (Short-term Scheduler)

  • 역할: 준비 큐에 있는 프로세스 중 어떤 프로세스를 다음에 실행할지 결정 (CPU 스케줄링)
  • 특징: 미리 정해진 스케줄링 알고리즘에 따라 프로세스를 선택하고 CPU에 디스패치(Dispatch) 함

3. CPU 스케줄링 과정 

    [준비 큐 (Ready Queue)]
           │
           │ (1) 디스패치 (단기 스케줄러)
           ↓
    ┌─────────────┐
    │  실행 상태    │ ── (2) 완료 ──> [종료 (Terminated)]
    │  (Running)  │
    └─────────────┘
           │  ↑
           │  │ (3) 타임아웃 (타임 슬라이스 만료)
           │  └──────────────────────┐
           │                         │
           │ (4) I/O 요청/이벤트 대기    │
           ↓                         │
    ┌─────────────┐                  │
    │  대기 상태    │ ── (5) I/O 완료 ──┘
    │  (Waiting)  │
    └─────────────┘
  1. 디스패치: 스케줄러가 준비 큐의 프로세스 중 하나를 선택해 CPU 자원을 할당
  2. 종료: 프로세스 수행이 완료되면 시스템 자원을 반납하고 종료
  3. 타임아웃: 할당된 시간을 초과하면 인터럽트가 발생하여 다시 준비 큐로 이동 (선점형)
  4. 블록(Block): I/O 요청 등이 발생하면 대기 큐로 이동하여 이벤트가 끝날 때까지 대기
  5. 웨이크업(Wake-up): 대기하던 I/O 작업이 완료되면 다시 준비 큐로 진입하여 차례를 기다림

4. 스케줄링 알고리즘 평가 기준

  • CPU 사용률 (CPU Utilization): 전체 시스템 시간 중 CPU가 실제로 가동된 시간의 비율 (최대화)
  • 처리량 (Throughput): 단위 시간당 완료(종료)된 프로세스의 수 (최대화)
  • 반환 시간 (Turnaround Time): 프로세스가 생성되어 종료될 때까지 걸린 총 시간 (최소화)
  • 대기 시간 (Waiting Time): 프로세스가 준비 큐에서 CPU 할당을 기다린 시간의 총합 (최소화)
  • 응답 시간 (Response Time): 사용자 요청 후 첫 번째 결과가 나올 때까지 걸린 시간 (최소화)

5. 스케줄링 알고리즘의 분류

비선점형 스케줄링 (Non-preemptive)

한 프로세스가 CPU를 점유하면 종료되거나 스스로 대기 상태로 전환될 때까지 CPU를 뺏을 수 없는 방식

알고리즘 특징
FCFS (First-Come, First-Served) 먼저 도착한 순서대로 처리. 실행 시간이 긴 프로세스가 앞에 있으면 효율이 떨어지는 호위 효과(Convoy Effect) 발생
SJF (Shortest Job First) 실행 시간이 짧은 프로세스 우선 처리. 평균 대기 시간이 가장 짧지만, 긴 작업이 계속 밀리는 기아 현상 가능
HRRN (Highest Response Ratio Next) 대기 시간과 실행 시간을 모두 고려하여 우선순위 산정 (SJF의 기아 현상 보완)

선점형 스케줄링 (Preemptive)

운영체제가 강제로 실행 중인 프로세스의 CPU 점유를 중단시키고 다른 프로세스에 할당할 수 있는 방식

알고리즘 특징
RR (Round Robin) 각 프로세스에 동일한 타임 슬라이스를 할당. 응답 속도가 빠르지만, 너무 짧으면 문맥 교환 오버헤드가 커짐
SRTF (Shortest Remaining Time First) 잔여 실행 시간이 가장 짧은 프로세스 우선 처리 (SJF의 선점형 버전)
MLQ (Multi-Level Queue) 준비 큐를 여러 개로 나누어 각기 다른 알고리즘 적용 (예: 시스템/대화형/배치 프로세스 분리)
MLFQ (Multi-Level Feedback Queue) 프로세스가 큐 사이를 이동할 수 있게 하여 MLQ의 경직성을 해결하고 유연한 우선순위 부여
