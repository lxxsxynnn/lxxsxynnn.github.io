데이터베이스 기초, NoSQL, 복잡 쿼리, ORM 실습 종합 정리

1. 데이터베이스란?

데이터를 체계적으로 저장·관리·검색할 수 있는 정보 저장소.

  • RDB(Relational, 관계형)와 NoSQL(비관계형) 등 다양한 유형 존재.

2. 데이터베이스 주요 용어

용어 설명
DBMS 데이터베이스 관리 시스템 (Oracle, MySQL 등)
테이블 데이터를 행/열로 저장하는 구조 단위
레코드/row 한 행에 해당하는 데이터 집합
필드/column 한 열에 해당하는 데이터 속성
쿼리(Query) 데이터를 조회/추가/수정/삭제하는 명령문
스키마 데이터베이스 구조와 설계 정보
인덱스 조회 성능 향상을 위한 데이터 구조
뷰(View) 결과 집합을 테이블처럼 보여주는 가상 테이블
트랜잭션 일관성 보장 작업 단위(ACID 특징)

3. SQL 기본문법 및 실습 예제 

-- 테이블 생성
CREATE TABLE member (
    id INT PRIMARY KEY,
    name VARCHAR(50),
    age INT
);

-- 삽입
INSERT INTO member (id, name, age) VALUES (1, 'Alice', 22);

-- 조회
SELECT id, name, age FROM member;

-- 수정
UPDATE member SET age = 23 WHERE id = 1;

-- 삭제
DELETE FROM member WHERE id = 1;

4. 트랜잭션과 ACID

  • 원자성(Atomicity): 전체 작업 성공/실패 단일 처리
  • 일관성(Consistency): 트랜잭션 전후 DB 상태 일관
  • 격리성(Isolation): 동시에 여러 트랜잭션 충돌없이 동작
  • 지속성(Durability): 작업 결과는 영구 저장

트랜잭션 예시 

BEGIN;
UPDATE account SET balance = balance - 100 WHERE id = 1;
UPDATE account SET balance = balance + 100 WHERE id = 2;
COMMIT;
-- 실패시 ROLLBACK 가능

5. 인덱스와 성능

  • 인덱스는 데이터 탐색 성능 증대 (빠른 조회)
  • 과도한 인덱스는 쓰기 성능 저하, 히트율 관리가 중요
  • 인덱스 종류: B-tree, 해시 등
  • 튜닝 시 인덱스 활용도(조회/삽입/전체 스캔 빈도) 반드시 분석

6. 정규화와 설계

  • 정규화: 데이터 중복 최소화, 무결성 유지(1NF, 2NF, 3NF 등 단계적 설계)
  • 과도한 정규화는 쿼리 복잡성 증가(JOIN 과다)
  • 실전에서는 업무/성능 기준 반정규화 필요

7. 복잡 쿼리 실습 예제 

-- 여러 테이블 JOIN 및 집계
SELECT u.name, SUM(o.price) AS total_price
FROM users u
JOIN orders o ON u.id = o.user_id
WHERE o.status = 'COMPLETE'
GROUP BY u.name
HAVING SUM(o.price) > 1000
ORDER BY total_price DESC;

-- 서브쿼리 및 윈도우함수 예시
SELECT name, age, RANK() OVER (ORDER BY age DESC) as rank
FROM member
WHERE age > (SELECT AVG(age) FROM member);

8. NoSQL vs RDBMS 비교

항목 RDBMS NoSQL (MongoDB 등)
구조 테이블(행/열) 컬렉션/문서 등(유연함)
스키마 엄격한 스키마(정형) 느슨하거나 무스키마(비정형)
트랜잭션 ACID 완전 지원 제한적/BASE 원칙
확장성 수직적(Server 늘리기) 수평적(Shard/Replica 분산)
용도 예시 금융, ERP, 통계 등 채팅, 로그, IoT 실시간 저장
쿼리언어 SQL 다양한 API/클라이언트(Query, Find 등)
정규화 필수, 데이터 중복 최소화 필요에 따라 중복 허용

NoSQL 예제(MongoDB) 

db.users.insertOne({ name: 'Alice', age: 22, tags: ['student', 'admin'] })
db.users.find({ age: { $gt: 20 } })
db.users.updateOne({ name: 'Alice' }, { $set: { age: 23 } })
db.users.deleteOne({ name: 'Alice' })

9. ORM 실습 예제(Java JPA) 

// 엔티티 클래스 예시

import javax.persistence.*;

@Entity
public class Member {
    @Id
    private Long id;
    private String name;
    private int age;
    // Getter/Setter 생략
}

// 쿼리/저장 예시
Member m = new Member();
m.setId(1L);
m.setName("Alice");
m.setAge(22);
entityManager.persist(m);

// JPQL 조회
List<Member> result = entityManager
  .createQuery("SELECT m FROM Member m WHERE m.age > 20", Member.class)
  .getResultList();

10. DB 구조 및 흐름 도식 

[클라이언트 요청]
  ↓
[커넥션 & 트랜잭션 관리]
  ↓
[SQL/NoSQL 파싱·실행]
  ↓
[테이블/컬렉션 & 인덱스 조회/변경]
  ↓
[로그/저널 생성]
  ↓
[데이터 영구 저장 및 응답 반환]

11. 실무/면접 핵심 팁

  • 트랜잭션 분리레벨 요구/충돌 사례(Dirty Read, Lost Update 등)
  • 인덱스/조회 성능, 튜닝 쿼리(EXPLAIN, AUTOTRACE 활용)
  • DB 설계 시 관계, 제약조건, 보안관리(NS, 권한체계) 고려
  • 분산 DB/파티셔닝, 샤딩 구조 실전 적용 경험
  • ORM(JPA, MyBatis) 활용 이유와 한계(복잡 쿼리 vs 직접 SQL)
