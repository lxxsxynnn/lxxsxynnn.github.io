[CS 스터디] 메모리 관리 전략

2026-06-04

1. 논리 메모리와 물리 메모리

운영체제는 프로세스가 직접 하드웨어 주소에 접근하지 못하도록 가상의 주소 공간을 제공함

  • 논리 주소 (Logical Address): 프로세스가 생성될 때 부여되는 고유한 주소 영역으로, CPU가 프로세스를 실행하며 참조함
  • 물리 주소 (Physical Address): 실제 메모리(RAM) 칩에 존재하는 물리적인 위치 주소
  • 주소 바인딩 (Address Binding): 프로그램의 논리 주소를 물리적 메모리 주소로 연결하는 과정

MMU (Memory Management Unit)

CPU가 메모리에 접근하기 전에 논리 주소를 물리 주소로 실시간 변환해주는 하드웨어 장치

    [ CPU ] ── 논리 주소 (300) ──> [ MMU ] ── 물리 주소 (14300) ──> [ RAM ]
                                     ↑
                             재배치 레지스터 (14000)
                             (시작 주소 저장)
  • 역할: 주소 변환뿐만 아니라, 프로세스가 허용되지 않은 메모리 영역(다른 프로세스나 OS 영역)에 접근하지 못하도록 보호하는 역할도 수행함

2. 연속 메모리 할당

메모리의 연속적인 공간에 프로세스를 통째로 로드하는 방식

고정 분할 방식 (Fixed Partitioning)

메모리를 미리 고정된 크기의 파티션으로 나누어 둠

  • 특징: 관리가 매우 쉽지만, 프로세스 크기가 파티션보다 작으면 공간이 남고(내부 단편화), 크면 들어가지 못하는(외부 단편화) 문제가 발생함

가변 분할 방식 (Variable Partitioning)

프로세스가 로드될 때 그 크기에 딱 맞춰 메모리를 할당함

  • 동적 할당 문제: 프로세스의 생성과 종료가 반복되면 메모리 사이에 사용 불가능한 작은 조각들이 생김 (외부 단편화)

메모리 할당 전략:

  1. 최초 적합 (First-fit): 가용 공간 중 프로세스를 수용할 수 있는 가장 첫 번째 공간에 할당 (가장 빠름)
  2. 최적 적합 (Best-fit): 가용 공간 중 프로세스 크기와 가장 차이가 적은 공간에 할당 (공간 효율은 좋으나 전체 탐색 오버헤드 발생)
  3. 최악 적합 (Worst-fit): 가장 큰 가용 공간에 할당 (남겨진 공간을 다시 의미 있게 쓰기 위함이지만 효율은 낮음)

메모리 압축 (Compaction): 외부 단편화를 해결하기 위해 현재 실행 중인 프로세스들을 한데 모아 가용 공간을 하나로 합치는 방법. 시스템 자원 소모가 매우 큼

3. 비연속 메모리 할당 (페이징과 세그멘테이션)

현대 운영체제는 외부 단편화 문제를 해결하기 위해 프로세스를 쪼개어 저장하는 방식을 사용함

페이징 (Paging)

프로세스의 논리 메모리를 고정된 크기인 **페이지(Page)**로 나누고, 물리 메모리를 동일한 크기의 **프레임(Frame)**으로 나누어 관리함

    [ 논리 메모리 ]        [ 페이지 테이블 ]        [ 물리 메모리 ]
    ┌──────────┐          ┌──────┬──────┐          ┌──────────┐
    │ 페이지 0   │ ── 0 ──> │  0   │  5   │ ── 5 ──> │ 프레임 5  │
    ├──────────┤          ├──────┼──────┤          ├──────────┤
    │ 페이지 1   │ ── 1 ──> │  1   │  2   │ ── 2 ──> │ 프레임 2  │
    ├──────────┤          ├──────┼──────┤          ├──────────┤
    │ 페이지 2   │ ── 2 ──> │  2   │  8   │ ── 8 ──> │ 프레임 8  │
    └──────────┘          └──────┴──────┘          └──────────┘
  • 장점: 물리적 메모리에 연속적으로 올릴 필요가 없어 외부 단편화가 완벽히 해결됨
  • 단점: 프로세스 크기가 페이지 크기의 배수가 아닐 경우 마지막 페이지에서 내부 단편화가 발생할 수 있음
  • 페이지 테이블 구조:
    • 계층적 페이징: 페이지 테이블이 너무 커지는 것을 방지하기 위해 테이블 자체를 다시 페이징함
    • 역 페이지 테이블: 프로세스마다 테이블을 두는 대신, 물리 메모리의 프레임마다 어떤 프로세스의 페이지가 있는지 기록하여 메모리 절약

세그멘테이션 (Segmentation)

프로세스를 논리적인 의미 단위인 세그먼트(Segment)(Code, Data, Stack, Main 등)로 분할함

    [ 논리 메모리 ]        [ 세그먼트 테이블 ]       [ 물리 메모리 ]
    ┌──────────┐          ┌──────┬──────┬──────┐    ┌──────────┐
    │  Main    │ ── 0 ──> │ 번호  │ limit│ base │    │  Stack   │
    ├──────────┤          ├──────┼──────┼──────┤    ├──────────┤
    │  Data    │ ── 1 ──> │  0   │ 1000 │ 2000 │    │          │
    ├──────────┤          ├──────┼──────┼──────┤    ├──────────┤
    │  Stack   │ ── 2 ──> │  1   │ 500  │ 4500 │    │  Main    │
    └──────────┘          └──────┴──────┴──────┘    └──────────┘
  • 장점: 의미 단위로 나누어져 있어 공유(Shared Code) 및 보안 설정에 매우 유리함
  • 단점: 세그먼트의 크기가 제각각이므로 할당/해제 반복 시 다시 외부 단편화 문제가 발생함

4. 단편화(Fragmentation) 상세 비교

단편화는 메모리가 낭비되어 프로세스를 수용하지 못하는 모든 현상을 일컬음

  1. 내부 단편화 (Internal Fragmentation)
    • 원인: 프로세스에게 할당된 메모리 조각이 실제 프로세스보다 커서 내부 공간이 남는 경우
    • 발생: 고정 분할 방식, 페이징(마지막 페이지)
  2. 외부 단편화 (External Fragmentation)
    • 원인: 가용 메모리 전체 합은 충분하지만, 연속적이지 않고 조각나 있어 프로세스를 올리지 못하는 경우
    • 발생: 가변 분할 방식, 세그멘테이션
