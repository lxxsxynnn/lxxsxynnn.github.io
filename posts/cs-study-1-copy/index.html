2026-06-01

1. 운영체제란

운영체제(Operating System, OS) : 하드웨어 바로 위에 설치되어, 하드웨어 계층과 다른 소프트웨어 계층을 연결하는 핵심 소프트웨어 계층

  • 역할: 컴퓨터 시스템의 자원(CPU, 메모리, 입출력 장치 등)을 효율적으로 관리하고, 사용자가 컴퓨터를 편리하게 사용할 수 있는 환경을 제공
  • 예시: Windows, macOS, Linux, Unix 등
      [사용자]
         ↓
    [응용 프로그램]
         ↓
      [유틸리티]
         ↓
     ["운영체제"] ← 하드웨어와 소프트웨어의 가교 역할
         ↓
      [하드웨어]

2. 운영체제의 목적

운영체제는 한정된 자원 안에서 정확하고 빠르게 문제를 해결하는 것을 목표로 함

주요 평가 지표

  1. 처리 능력 향상 (Throughput): 일정 시간 내에 시스템이 처리하는 일의 양을 최대화
  2. 반환 시간 단축 (Turnaround Time): 사용자가 작업을 요청한 시점부터 완료될 때까지 걸리는 시간을 최소화
  3. 가용성 향상 (Availability): 사용자가 필요할 때 즉시 자원을 사용할 수 있도록 관리
  4. 신뢰도 향상 (Reliability): 입력된 값에 대해 항상 정확하고 일관된 결과를 제공

3. CPU와 메모리 구조

컴퓨터의 핵심은 연산을 담당하는 CPU와 데이터를 저장하는 메모리

CPU (Processor)

컴퓨터에서 프로그램을 실행하는 데 필요한 복잡한 연산을 처리하고 명령을 수행하는 ‘두뇌’ 역할

메모리 (Memory)

데이터를 저장하기 위한 기억장치로, 속도와 용도에 따라 계층 구조를 가짐

  • 주기억장치: RAM 등 휘발성 메모리로, CPU가 직접 접근하여 데이터를 읽고 씀
  • 보조기억장치: SSD, HDD 등 비휘발성 메모리로, 데이터를 영구적으로 저장함

메모리 계층 구조 (상단일수록 속도가 빠르고 용량이 작음)

    [  레지스터 ]  ──┐ CPU 내부
    [   캐시   ]  ──┘
    [   RAM   ]  ── 주기억장치
    [ 하드디스크 ]  ── 보조기억장치
  • 레지스터: CPU 내부에서 연산에 필요한 데이터를 아주 짧은 시간 동안 저장
  • 캐시 메모리: CPU와 RAM 사이의 속도 차이를 줄여 성능을 높이는 완충 공간
  • RAM: 프로그램 실행 시 필요한 정보를 로드하는 공간, 전원이 꺼지면 데이터가 사라지는 휘발성이 있음
  • 디스크(SSD/HDD): 운영체제, 프로그램, 사용자 데이터를 영구 저장하는 비휘발성 저장소

프로그램 실행 과정: 사용자가 프로그램을 실행하면, OS는 디스크의 프로그램을 메모리로 로드
이때 메모리에 로드되어 실행 중인 상태를 프로세스라고 함

4. 커널과 시스템 콜

커널(Kernel): 운영체제의 핵심부로, 하드웨어와 프로세스 사이에서 자원 관리 및 보안과 같은 중추적인 역할을 수행

  [응용 프로그램]
       ↕️
     [커널] (CPU 스케줄링, 메모리 관리, 입출력 제어 등)
   ↕️       ↕️       ↕️
 [CPU]   [메모리]  [외부기기]

모드 전환 (Mode Bit)

운영체제는 시스템의 중요한 자원을 보호하기 위해 실행 모드를 두 가지로 분리함

  • 커널 모드 (Kernel Mode): 하드웨어에 직접 접근하여 모든 자원을 제어할 수 있는 모드
  • 사용자 모드 (User Mode): 일반 응용 프로그램이 실행되는 모드로, 시스템 자원에 직접 접근이 제한됨

시스템 콜 (System Call)

사용자 모드의 프로그램이 하드웨어 자원이나 커널의 기능이 필요할 때, 커널에 서비스를 요청하는 통로

  • 예시: fork() (프로세스 생성), wait() (자식 프로세스 대기), read()/write() (파일 조작)
  • 매개변수 전달 방식:
    1. 레지스터를 통해 직접 전달
    2. 메모리 주소값을 레지스터에 저장하여 전달
    3. 스택(Stack)을 이용해 전달
