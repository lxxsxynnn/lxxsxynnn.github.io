1. 요약

선형 자료구조

  • 데이터가 연속적으로 나열되는 자료구조
  • 종류:
    • 배열: 정해진 크기만큼 데이터가 일렬로 저장되는 정적 자료구조 (인덱스로 O(1) 접근)
    • 연결 리스트: 크기가 정해져 있지 않은 동적 자료구조로, 노드를 포인터로 연결해 구성
    • 스택: 마지막에 들어온 데이터가 먼저 나가는 후입선출(LIFO) 구조
    • 큐: 먼저 들어온 데이터가 먼저 나가는 선입선출(FIFO) 구조
    • 덱(Deque): 양쪽 끝에서 데이터 추가·삭제가 모두 가능한 구조

비선형 자료구조

  • 하나의 데이터 뒤에 N개의 데이터가 이어질 수 있는 자료구조
  • 계층적 구조를 표현하기 좋음
  • 종류: 그래프, 트리, 우선순위 큐, 힙, 해시 테이블

그래프

  • 데이터를 포함하는 정점(vertex) 과 정점을 잇는 간선(edge) 으로 구성
  • 사이클을 가질 수 있음
  • 탐색 방법:
    • BFS(너비 우선 탐색): 가까운 정점부터 넓게 탐색 (큐, 최단 거리)
    • DFS(깊이 우선 탐색): 한 방향으로 깊게 탐색 (스택·재귀, 경로 존재 여부)

트리

  • 사이클이 없는 그래프의 한 종류로, 계층 관계 를 표현할 수 있음
  • 이진 트리: 자식 노드가 최대 2개인 트리
    • 완전 이진 트리: 마지막 레벨을 제외한 모든 레벨이 채워져 있고, 마지막 레벨은 왼쪽부터 채워진 트리
    • 포화 이진 트리: 마지막 레벨까지 노드가 모두 채워진 트리
    • 이진 탐색 트리(BST): 왼쪽 서브트리는 부모보다 작은 값, 오른쪽 서브트리는 부모보다 큰 값 을 갖는 트리
    • 균형 이진 탐색 트리: 삽입·삭제 후에도 균형을 유지해 검색 O(log n) 을 보장하는 트리 (AVL 트리, 레드-블랙 트리)

힙

  • 최댓값 또는 최솟값을 빠르게 찾을 수 있는 완전 이진 트리
  • 최대 힙: 부모 노드 값 ≥ 자식 노드 값 → 루트가 최댓값
  • 최소 힙: 부모 노드 값 ≤ 자식 노드 값 → 루트가 최솟값

해시 테이블

  • 키(key) 에 대한 값(value) 을 저장하는 자료구조
  • 키를 해시 함수에 넣어 얻은 해시 를 인덱스로 사용해 값에 접근 → 평균 O(1)
  • 해시 충돌: 서로 다른 키가 같은 해시 값을 갖는 현상
  • 해결 방법:
    • 체이닝: 같은 해시 값을 갖는 데이터를 연결 리스트로 묶어 저장
    • 개방 주소법: 비어 있는 다른 공간에 저장 (선형 조사법, 이차 조사법, 이중 해싱)

2. 면접 대비 (Q&A)

배열과 연결 리스트의 차이점을 설명해 보세요.

배열은 연속된 메모리 공간에 정해진 크기만큼 데이터를 저장하는 정적 자료구조로, 인덱스를 통해 원하는 원소에 O(1)로 바로 접근할 수 있지만 중간 삽입·삭제 시 뒤의 원소들을 모두 이동해야 해 O(n)이 걸립니다.
연결 리스트는 노드를 포인터로 연결한 동적 자료구조로, 크기가 정해져 있지 않아 삽입·삭제가 O(1)로 빠르지만 특정 원소에 접근하려면 처음부터 순차 탐색해야 하므로 O(n)이 걸립니다.

단순 연결 리스트를 역순으로 출력하는 방법을 설명해 보세요.

단순 연결 리스트는 한 방향으로만 연결되어 있어 직접 거슬러 올라갈 수 없으므로 크게 세 가지 방법을 사용합니다.

  • 재귀 호출: 노드를 끝까지 따라간 뒤, 재귀가 풀리는 순서대로 값을 출력하면 역순이 됩니다.
  • 스택 이용: 앞에서부터 순회하며 각 노드 값을 스택에 쌓은 뒤, 스택을 비우면서 출력하면 LIFO 특성상 역순으로 나옵니다.
  • 링크 뒤집기: prev·curr·next 포인터로 각 노드의 연결 방향을 반대로 뒤집어 리스트 자체를 역순으로 만든 뒤 출력합니다.
스택은 어떤 경우에 주로 사용하나요?

스택은 마지막에 들어온 데이터가 먼저 나가는 LIFO 구조라서, ‘최근 것’을 먼저 되돌리거나 처리해야 하는 상황에 주로 사용됩니다.
대표적으로 함수 호출을 관리하는 콜 스택, 실행 취소(undo) 기능, 괄호 짝 검사, 후위 표기법 수식 계산, 그리고 깊이 우선 탐색(DFS) 등에 활용됩니다.

큐를 스택으로 구현하는 방법을 설명해 보세요.

스택 2개(입력용·출력용)를 이용해 FIFO 동작을 구현할 수 있습니다.
데이터를 넣을 때(enqueue)는 입력 스택에 그대로 push 합니다.
데이터를 꺼낼 때(dequeue)는 출력 스택이 비어 있으면 입력 스택의 모든 원소를 꺼내 출력 스택으로 옮겨 담습니다. 이때 순서가 한 번 뒤집히므로 출력 스택에서 pop 하면 가장 먼저 들어온 데이터부터 나와 큐처럼 동작하게 됩니다.

우선순위 큐는 무엇인가요?

우선순위 큐는 들어온 순서와 관계없이 우선순위가 가장 높은 데이터가 먼저 나오는 자료구조입니다.
일반 큐처럼 삽입·삭제 연산을 지원하지만, 삭제 시 우선순위가 가장 높은 데이터가 나온다는 점이 다릅니다.
배열이나 연결 리스트로도 구현할 수 있지만 삽입·삭제가 모두 O(log n)으로 효율적인 힙으로 구현하는 것이 일반적입니다.

힙이란 무엇인가요?

힙은 최댓값이나 최솟값을 빠르게 찾을 수 있도록 만들어진 완전 이진 트리 형태의 자료구조입니다.
부모 노드의 값이 자식 노드보다 항상 크거나 같으면 루트가 최댓값이 되는 최대 힙, 부모 노드가 자식보다 항상 작거나 같으면 루트가 최솟값이 되는 최소 힙입니다.
삽입·삭제가 O(log n)으로 처리되어 우선순위 큐를 구현하는 데 자주 사용됩니다.

이진 탐색 트리, 포화 이진 트리, 완전 이진 트리의 차이점은 무엇인가요?

셋 다 자식 노드가 최대 2개인 이진 트리이지만 기준이 다릅니다.

  • 완전 이진 트리: 마지막 레벨을 제외한 모든 레벨이 채워져 있고, 마지막 레벨은 왼쪽부터 오른쪽으로 채워진 트리입니다.
  • 포화 이진 트리: 마지막 레벨까지 모든 노드가 빈틈없이 채워진 트리로, 완전 이진 트리에도 포함됩니다.
  • 이진 탐색 트리(BST): 노드 값에 규칙이 있어 왼쪽 서브트리는 부모보다 작은 값, 오른쪽 서브트리는 부모보다 큰 값으로 구성된 트리입니다.
즉, 완전·포화 이진 트리는 노드가 채워진 '모양'에 대한 분류이고, 이진 탐색 트리는 노드 값의 '정렬 규칙'에 대한 분류입니다.
이진 탐색 트리에서 발생할 수 있는 문제는 무엇이고, 이를 보완할 수 있는 방법이 있다면 무엇인가요?

이진 탐색 트리는 데이터가 한쪽으로 치우쳐 삽입되면 트리의 균형이 깨져 한쪽으로 길게 늘어진 편향 트리가 될 수 있습니다.
이 경우 사실상 연결 리스트와 같아져 검색 시간이 O(log n)이 아닌 O(n)으로 늘어나 트리의 장점이 사라집니다.
이를 보완하기 위해 삽입·삭제 시 항상 균형을 맞춰 O(log n)을 보장하는 균형 이진 탐색 트리를 사용합니다. 대표적으로 좌·우 서브트리의 높이 차를 1 이하로 유지하는 AVL 트리, 노드에 색을 칠해 느슨하게 균형을 유지하는 레드-블랙 트리가 있습니다.

트리와 그래프의 차이점은 무엇인가요?

트리는 그래프의 한 종류로, 사이클이 없고 계층 관계를 표현하는 그래프입니다.
그래프는 정점과 간선으로 이루어져 사이클이 존재할 수 있고 방향성·가중치 등을 가질 수 있으며 루트 개념이 없습니다.
반면 트리는 사이클이 없고 부모-자식의 방향성을 가지며, 하나의 루트 노드에서 시작해 정점이 N개일 때 간선이 항상 N-1개라는 특징이 있습니다.

BFS와 DFS의 차이점은 무엇인가요?

둘 다 그래프의 모든 정점을 탐색하는 방법이지만 탐색 순서가 다릅니다.
BFS(너비 우선 탐색)는 시작 정점에서 가까운 정점부터 레벨 단위로 넓게 탐색하며 큐로 구현합니다. 비가중치 그래프에서 시작 정점부터의 최단 거리를 구할 수 있습니다.
DFS(깊이 우선 탐색)는 한 방향으로 갈 수 있는 데까지 깊게 내려간 뒤 막히면 되돌아오며 탐색하고, 스택이나 재귀로 구현합니다. 두 정점 간 경로의 존재 여부를 확인하는 데 적합하며 재귀로 구현하면 저장 공간을 적게 사용합니다.

해시 테이블이란 무엇인가요?

해시 테이블은 하나의 키에 하나의 값을 저장하는 자료구조입니다.
키를 해시 함수에 넣어 얻은 해시 값을 인덱스로 사용해 값을 저장·조회하기 때문에, 원하는 값에 평균 O(1)로 한 번에 접근할 수 있습니다.
다만 서로 다른 키가 같은 해시 값을 갖는 해시 충돌이 발생할 수 있어, 체이닝이나 개방 주소법으로 이를 해결합니다.

해시 테이블을 이용할 때 해시 충돌이 발생할 경우 어떻게 처리할 수 있나요?

크게 체이닝과 개방 주소법 두 가지 방법이 있습니다.

  • 체이닝(Chaining): 같은 해시 값을 갖는 데이터들을 연결 리스트로 묶어 저장합니다. 저장 공간 제약이 적지만 한 해시에 데이터가 몰리면 그 부분의 탐색이 느려집니다.
  • 개방 주소법(Open Addressing): 충돌이 발생하면 다른 빈 공간을 찾아 저장합니다. 한 칸씩 옮기는 선형 조사법, 제곱 간격으로 옮기는 이차 조사법, 또 다른 해시 함수로 이동 폭을 정하는 이중 해싱이 있습니다.
