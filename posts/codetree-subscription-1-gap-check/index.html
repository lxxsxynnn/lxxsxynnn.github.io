단순히 문제를 푸는 것을 넘어, ‘내가 무엇을 모르는지’를 아는 것이 얼마나 중요한지 깨달은 시간이었습니다.

🔍 갭체크를 하며 느낀 점: “아는 것과 구현하는 것의 차이”

처음에는 “그래도 구현 문제 정도는 어느 정도 풀 수 있지 않을까?” 라고 자신했지만, 실전은 달랐습니다. 생각보다 훨씬 디테일한 부분에서 발목을 잡혔고, 문제 난이도는 입문~중급 수준이었지만 제한된 시간 내에 완벽히 구현해내는 것이 쉽지 않았습니다.

1. 원형 바구니 시뮬레이션

가장 기억에 남는 문제입니다. 단순히 반복문을 돌리면 될 것 같았지만, 아래 조건들이 얽히니 로직이 금세 꼬여버렸습니다.

  • 시작 위치 갱신 및 시계/반시계 방향 처리
  • 자기 자신 포함 여부에 따른 예외 처리
  • 원형 인덱스 처리 (% n)

💡 내 풀이 vs 더 나은 풀이 (코드 비교)

❌ 나의 초기 풀이 (복잡한 인덱스 계산)

반복문 안에서 start 위치를 수동으로 계산하려다 보니 b-1 같은 인덱스 처리에서 실수가 잦았고, 반시계 방향 처리가 매우 까다로웠습니다.

// 시뮬레이션 내 풀이 (초기 버전)
for (int i = 0; i < q; i++) {
    int a = sc.nextInt(); // 이동 방향
    int b = sc.nextInt(); // 이동 횟수
    if(a == 1) {
        for(int j = start; j < b - 1; j++){
            balls[j]++;
            start += b - 1;
        }
    } else {
        // 반시계 방향 로직 구현 중 오류 발생
    }
}

✅ 개선된 풀이 (나머지 연산 활용)

나머지 연산(% n)을 활용하니 원형 구조를 단 한 줄로, 어떤 방향이든 안전하게 처리할 수 있었습니다.

// 개선된 풀이 (Modulo 활용)
for (int i = 0; i < q; i++) {
    int a = sc.nextInt(); // 방향 (1 or -1)
    int b = sc.nextInt(); // 이동 횟수

    for (int j = 0; j < b; j++) {
        // (현재위치 + 방향 + 전체길이) % 전체길이 로 원형 인덱스 해결
        start = (start + a + n) % n;
        balls[start]++;
    }
}

2. 비트마스크와 완전탐색

‘음료수 문제’는 아예 접근조차 못 했습니다. “N개를 선택한다"는 조건을 보고 조합이나 비트마스크를 즉각적으로 떠올리지 못했습니다. 나중에 보니 비트마스크를 활용한 완전탐색 문제였는데, 특정 유형에 대한 경험치 부족을 뼈저리게 실감했습니다.

📊 분석 결과: 나의 강점과 약점 리포트

분석 결과는 냉정하지만 정확했습니다.

  • 강점: 기본적인 제어문 사용과 간단한 탐색 로직은 이해하고 있었습니다.
  • 약점: **시뮬레이션 I (불안정)**과 완전탐색 I (부족).
    • 시뮬레이션: 문법의 문제가 아니라, 문제를 끝까지 정확하게 설계하고 구현하는 ‘지구력’이 부족했습니다.
    • 완전탐색: 전체 경우의 수를 판단하고 탐색 범위를 결정하는 감각이 부족했습니다.

✨ 코드트리 갭체크의 좋았던 점

다른 학습 서비스와 비교했을 때 코드트리만의 확실한 장점이 있었습니다.

  • 구체적인 진단과 학습 경로: 단순히 “맞았다/틀렸다"가 아니라, “시뮬레이션은 불안정”, “완전탐색은 부족"처럼 상태를 세분화하여 보여주고, 그에 맞는 추천 학습 경로를 바로 제시해 줍니다.
  • 이정표 제시: 막연하게 “코테가 어렵다"고 느끼던 상태에서, **“어디서부터 다시 시작해야 할지”**가 보이니 공부 의욕이 더 생겼습니다.

🎯 앞으로의 공부 방향

이번 갭체크를 통해 저의 현재 위치를 객관적으로 마주할 수 있었습니다. 당분간은 아래 두 가지에 집중하려 합니다.

  1. 구현 실수 제로화: 시뮬레이션 문제를 반복 풀이하며 조건 처리 능력 향상
  2. 완전탐색 패턴 익히기: 조합, 비트마스크 등 완전탐색의 정석 패턴 체득
  3. 정확도 우선: 빨리 푸는 것보다 “조건을 단 한 번에 정확하게 구현하는 연습”
