1. 조인이란

조인(Join): 여러 테이블을 합쳐 원하는 데이터를 얻는 작업
결과에 포함되는 테이블의 범주에 따라 내부 조인(Inner Join)과 외부 조인(Outer Join) 등으로 나뉨

조인 연산자

  • ON: 조인 연산에 대한 조건절을 작성할 때 사용하는 연산자
    각 테이블에서 어떠한 속성을 사용해 연산을 수행할지 나타냄
  • USING: 조인 연산을 하려는 테이블에서 속성명이 동일할 때 USING 연산자 뒤에
    공통되는 속성명만 작성해 조건을 간단하게 적을 수 있음
  • WHERE: 테이블에서 원하는 데이터를 조회하기 위한 조건절을 작성할 때 사용하는 연산자
    조건절에 비교 연산자, IN 연산자, BETWEEN 연산자 등을 사용해 원하는 데이터를 필터링할 수 있음
  • IN: WHERE 절과 함께 사용하며, IN 연산자 뒤에 나오는 조건에 하나라도 해당하는 값을 검색함
  • BETWEEN: 특정 범위에 속하는 데이터를 조회할 때 사용하는 연산자 (비교 연산자 >, < 등과 유사)
  • LIKE: 특정 패턴을 가진 데이터를 조회할 때 사용하는 연산자
    %(길이를 알 수 없는 문자), _(특정 길이의 문자)와 함께 사용

2. 내부 조인 (Inner Join)

2개 이상의 테이블에서 공통 속성을 가진 데이터를 검색하기 위한 연산

SELECT [조회하려는 열]
FROM [테이블1] INNER JOIN [테이블2]
ON [조인 조건]
WHERE [검색 조건]
  • 명시적 조인: JOIN 연산자 뒤에 조인할 테이블을 명시적으로 표현하는 방법
  • 암시적 조인: JOIN 연산자를 명시하지 않고 FROM 절 뒤에 조인할 테이블만 나열하는 방법
-- 암시적 조인 예시
SELECT *
FROM [테이블1], [테이블2]
WHERE [조인 조건]

3. 외부 조인 (Outer Join)

조인 조건에 해당하는 속성 값이 없는 데이터까지 모두 조회하는 연산
존재하지 않는 데이터에 대해서는 NULL로 표기함

  • 왼쪽 외부 조인 (Left Outer Join): 연산자 왼쪽 테이블의 모든 데이터를 결과 테이블에 포함
    조건을 만족하는 튜플이 왼쪽 테이블에만 있어도 결과에 포함됨
  • 오른쪽 외부 조인 (Right Outer Join): 연산자 오른쪽 테이블의 모든 데이터를 결과 테이블에 포함
    조건을 만족하는 튜플이 오른쪽 테이블에만 있어도 결과에 포함됨
  • 완전 외부 조인 (Full Outer Join): 양쪽 테이블에서 조건을 만족하지 않는 데이터가 있더라도 모두 포함해 결과 테이블에 나타냄
SELECT [조회하려는 열]
FROM [테이블1] [LEFT | RIGHT | FULL] OUTER JOIN [테이블2]
ON [조인 조건]

교차 조인 (Cross Join)

**카티션 곱(Cartesian Product)**이라고도 함
각 테이블의 행과 다른 테이블의 모든 행을 결합해 결과 테이블을 생성함 (별도의 조인 조건이 없어 모든 경우의 수가 결합됨)

SELECT *
FROM [테이블1] CROSS JOIN [테이블2]

셀프 조인 (Self Join)

한 테이블이 자기 자신과 조인 연산을 수행하는 것
FROM 절에 같은 테이블을 2번 적어야 하므로, 테이블을 구분하기 위해 반드시 **별칭(Alias)**을 사용함

SELECT *
FROM [테이블1] [별칭1], [테이블1] [별칭2]
WHERE [검색 조건]
