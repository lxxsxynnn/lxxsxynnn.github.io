[CS 스터디] 정렬 알고리즘 2

2026-06-29

1. 퀵 정렬 (Quick Sort)

  • 비교 기반의 분할 정복 알고리즘
  • 피봇(pivot) 이라는 기준 값을 정해, 피봇보다 작은 값 과 큰 값 으로 배열을 분할하며 정렬
  • 피봇은 보통 배열의 첫 번째 또는 마지막 요소 를 선택

분할 과정 — low·high 두 포인터로 작은 값과 큰 값을 분리

  • low: 첫 요소에서 시작해 오른쪽으로 이동, 피봇보다 큰 값 을 만나면 멈춤
  • high: 마지막 요소에서 시작해 왼쪽으로 이동, 피봇보다 작은 값 을 만나면 멈춤
  • 둘 다 멈추면 두 값을 교환 → 왼쪽엔 작은 값, 오른쪽엔 큰 값이 모임
  • 배열 크기가 1 이하 가 될 때까지 각 부분에 대해 반복
   피봇 = 5 (첫 요소)
   [5, 3, 8, 4, 1, 9, 2, 7, 6]
        │  피봇 기준으로 분할
        ▼
   [3, 4, 1, 2]   (5)   [8, 9, 7, 6]
    피봇보다 작음     │     피봇보다 큼
                   ▼
   각 부분 배열에 같은 과정을 재귀적으로 반복
   → [1, 2, 3, 4, 5, 6, 7, 8, 9]
  • 평균 시간 복잡도 O(n log n) — 피봇을 기준으로 배열이 균등하게 분할 될 때 성립
  • 최악 시간 복잡도 O(n²) — 한쪽으로만 분할되는 경우
    • 예) 역순 배열 [5, 4, 3, 2, 1]에서 첫 요소 5를 피봇으로 잡으면 더 큰 값이 없어 분할이 안 됨
  • 결국 피봇 선택이 성능을 좌우 → 중앙값이나 무작위 피봇으로 최악을 완화

2. 힙 정렬 (Heap Sort)

  • 최대 힙 / 최소 힙 자료구조를 이용하는 정렬
  • 두 단계로 나뉨
    1. 힙 생성: 배열을 힙으로 만듦
    2. 정렬: 힙에서 요소를 하나씩 꺼내며 정렬
  • 힙 생성 알고리즘: 특정 노드를 두 자식 중 우선순위가 높은 자식 과 비교·교환하며 힙 조건을 맞춤
   ① 배열 [1, 3, 2, 4]를 최대 힙으로 구성
            (4)
           /   \
         (3)    (2)
         /
       (1)

   ② 루트(최댓값)를 맨 뒤로 보내고 힙에서 제외 → 다시 힙 정렬(down-heap)
      4 추출 → 3 추출 → 2 추출 → 1
   → [1, 2, 3, 4]
  • 힙 생성(down-heap)에 O(log n) , n개 요소를 꺼내며 정렬해 O(n) → 전체 O(n log n)
  • 추가 배열 없이 제자리(in-place) 정렬이 가능함

3. 기수 정렬 (Radix Sort)

  • 값을 비교하지 않는 정렬 알고리즘
  • 낮은 자릿수부터 차례로 정렬을 수행
  • 각 자릿수(0~9)에 해당하는 버킷(큐) 를 만들어, 자릿수 숫자에 맞게 데이터를 분배·수집하며 자릿수만큼 반복
   배열 [41, 12, 34, 25, 9]

   ① 1의 자리로 분배 → 수집
      1:[41] 2:[12] 4:[34] 5:[25] 9:[09]
      → [41, 12, 34, 25, 9]

   ② 10의 자리로 분배 → 수집
      0:[09] 1:[12] 2:[25] 3:[34] 4:[41]
      → [9, 12, 25, 34, 41]   ← 정렬 완료
  • 데이터 개수 n, 최대 자릿수 d일 때 시간 복잡도 O(dn)
  • 단점: 버킷용 추가 메모리가 필요하고, 정렬 가능한 데이터 타입이 한정적

4. 계수 정렬 (Counting Sort)

  • 데이터의 개수를 세서 정렬하는 방식
  • 데이터 범위를 인덱스로 갖는 빈 배열을 만들고, 원본을 순회하며 해당 인덱스 값을 1씩 증가
  • 데이터 범위가 0 또는 양의 정수여야 한다는 제약이 있음
   배열 [2, 5, 3, 0, 2, 3, 0, 3]

   값 :   0   1   2   3   4   5
   개수:  2   0   2   3   0   1   ← 각 값의 등장 횟수

   개수만큼 순서대로 나열
   → [0, 0, 2, 2, 3, 3, 3, 5]
  • 데이터 개수 n, 최댓값 k일 때 시간 복잡도 O(n + k)
  • 최댓값 k가 커질수록 시간·공간이 함께 늘어남 → 데이터 범위만 한 배열 이 필요해 추가 메모리를 사용
  • 데이터 범위가 좁을 때 매우 빠름

정렬 알고리즘 비교

알고리즘 평균 시간 복잡도 최악 시간 복잡도 추가 메모리 특징
퀵 정렬 O(n log n) O(n²) O(log n) 피봇 선택이 성능을 좌우
힙 정렬 O(n log n) O(n log n) O(1) 제자리 정렬, 항상 O(n log n)
기수 정렬 O(dn) O(dn) O(n + 버킷) 비교 없이 자릿수로 정렬
계수 정렬 O(n + k) O(n + k) O(k) 데이터 범위가 좁을 때 빠름

퀵·힙 정렬은 비교 기반 , 기수·계수 정렬은 비교하지 않는 정렬이라 특정 조건에서 O(n log n)보다 빠를 수 있음
