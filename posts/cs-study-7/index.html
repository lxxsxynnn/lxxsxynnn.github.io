1. 요약

메모리 및 시스템 기초

  • 메모리 계층 구조: 속도, 용량, 비용의 효율성을 위해 계층화함 (레지스터 > 캐시 > RAM > HDD/SSD)
  • 캐시 메모리: CPU와 RAM 사이의 속도 차이를 극복하기 위한 고속 메모리
  • 시스템 콜: 프로세스가 운영체제 커널의 서비스를 요청하기 위한 인터페이스

프로세스와 스레드

  • 프로세스: 실행 중인 프로그램의 인스턴스
    독립된 메모리 공간(코드, 데이터, 힙, 스택)과 PCB를 가짐
  • 스레드: 프로세스 내 실행 흐름의 단위. 스택을 제외한 나머지 메모리 영역을 공유함
  • 멀티 프로세스 vs 멀티 스레드:
    • 멀티 프로세스: 안정성이 높지만 문맥 교환 오버헤드와 IPC 비용이 큼
    • 멀티 스레드: 자원 공유로 효율적이나 동기화 이슈(Race Condition) 주의 필요
  • 콘텍스트 스위칭: CPU 제어권을 넘길 때 현재 상태를 저장하고 새 상태를 로드하는 과정
    (스레드 간 전환이 프로세스 간 전환보다 빠름)

동기화 및 교착 상태

  • 프로세스 동기화: 공유 자원 접근 시 데이터 일관성을 보장하기 위한 기법(뮤텍스, 세마포어 등)
  • 교착 상태(Deadlock): 서로 자원을 점유한 채 대기하며 진행이 멈춘 상태
    • 4가지 발생 조건: 상호 배제, 점유와 대기, 비선점, 환형 대기

스케줄링 및 메모리 관리

  • CPU 스케줄링:
    • 비선점형: FCFS, SJF (뺏기 불가능)
    • 선점형: RR, SRTF (OS가 강제 회수 가능)
  • 메모리 할당:
    • 페이징: 고정 크기로 분할. 외부 단편화 해결, 내부 단편화 가능성
    • 세그먼테이션: 논리 단위로 분할. 내부 단편화 해결, 외부 단편화 가능성
  • 가상 메모리: 물리 메모리 한계를 극복하기 위해 보조 기억 장치를 활용하는 기술
    • 요구 페이징: 필요한 페이지만 메모리에 로드
    • 스레싱: 페이지 폴트 과다로 인해 CPU 사용률이 급감하는 현상

2. 면접 대비 (Q&A)

시스템 콜을 설명해보세요. 운영체제의 커널이 제공하는 서비스에 접근하기 위해 프로그램이 사용하는 인터페이스입니다.
프로세스 제어, 파일 및 장치 조작 등 시스템 자원을 안전하게 관리하기 위해 유저 모드에서 커널 모드로 전환할 때 사용됩니다.
프로세스의 메모리 구조를 설명해보세요. 프로세스는 크게 네 가지 영역으로 구성됩니다.
  • 코드(Code): 실행할 기계어 명령어가 저장되는 영역
  • 데이터(Data): 전역 변수, 정적 변수 등이 저장되는 영역
  • 힙(Heap): 런타임에 동적으로 할당되는 메모리 영역
  • 스택(Stack): 함수 호출 시 지역 변수, 매개변수, 복귀 주소 등이 저장되는 영역
프로세스와 스레드의 차이점을 설명해보세요. 프로세스는 운영체제로부터 자원을 할당받는 독립적인 실행 단위이며, 스레드는 프로세스 내에서 실행되는 여러 흐름의 단위입니다.
프로세스는 각각 독립된 메모리 공간을 가지지만, 스레드는 프로세스의 힙, 데이터, 코드 영역을 공유하고 자신만의 스택과 레지스터 정보만 가집니다.
스택 오버플로우와 힙 오버플로우에 관해 설명해보세요. 스택 오버플로우는 재귀 호출이 너무 깊어지거나 지역 변수가 너무 커서 스택 영역의 한계를 넘는 현상입니다.
힙 오버플로우는 사용자가 동적으로 할당한 메모리가 힙 영역의 범위를 초과하여 인접한 메모리 영역을 침범하는 현상입니다.
PCB가 무엇인지 설명해보세요. Process Control Block의 약자로, 운영체제가 각 프로세스를 관리하기 위해 유지하는 자료구조입니다.
프로세스 상태, PID, 프로그램 카운터(PC), 레지스터 값, 스케줄링 정보 등이 포함되어 있습니다.
멀티 프로세스와 멀티 스레드의 차이점을 설명해보세요. 멀티 프로세스는 각 프로세스가 독립적이어서 하나가 실패해도 다른 프로세스에 영향을 주지 않아 안정적이지만, 문맥 교환 비용이 크고 IPC를 통한 통신이 복잡합니다.
멀티 스레드는 자원을 공유하므로 통신이 빠르고 비용이 적지만, 하나의 스레드 장애가 전체 프로세스에 영향을 줄 수 있고 동기화 처리가 중요합니다.
동시성과 병렬성을 비교해 설명해보세요. 동시성(Concurrency)은 싱글 코어에서 여러 작업을 빠르게 번갈아 실행하여 '동시에 실행되는 것처럼 보이게' 하는 논리적인 개념입니다.
병렬성(Parallelism)은 멀티 코어에서 실제로 여러 작업을 '동시에' 처리하는 물리적인 개념입니다.
콘텍스트 스위칭이 무엇인지 설명해보세요. CPU가 현재 실행 중인 프로세스나 스레드의 상태를 저장하고, 다음으로 실행할 프로세스나 스레드의 상태를 복구하여 제어권을 넘기는 과정입니다.
이 과정에서 CPU는 아무런 일을 하지 못하므로 오버헤드가 발생합니다.
멀티 프로세서에서 콘텍스트 스위칭과 멀티 스레드에서 콘텍스트 스위칭의 차이점을 설명해보세요. 멀티 스레드 간의 콘텍스트 스위칭은 프로세스 내의 공유 자원(데이터, 힙 등)을 그대로 유지하고 스택과 레지스터만 교체하면 되므로,
주소 공간 자체를 바꿔야 하는 멀티 프로세스 간의 전환보다 오버헤드가 훨씬 적습니다.
프로세스 동기화에 대해 설명해보세요. 여러 프로세스나 스레드가 공유 자원에 동시에 접근할 때(Race Condition), 데이터의 일관성을 유지하기 위해 실행 순서를 조절하거나 자원 접근을 제한하는 것을 말합니다.
뮤텍스와 세마포어의 차이점을 설명해보세요. 뮤텍스는 오직 하나의 쓰레드만이 공유 자원을 점유할 수 있는 'Binary Lock' 개념입니다.
세마포어는 공유 자원에 접근할 수 있는 프로세스/스레드의 개수를 지정(Counter)할 수 있으며,
뮤텍스는 락을 건 스레드만 해제할 수 있지만 세마포어는 다른 프로세스에서도 신호를 줄 수 있습니다.
스레드 안전(Thread Safe)의 의미가 무엇이며 이를 구현하는 방법은 무엇인가요? 멀티 스레드 환경에서 여러 스레드가 동시에 함수나 변수에 접근해도 프로그램의 실행 결과가 올바르게 유지되는 상태를 말합니다.
이를 위해 Mutex나 Semaphore를 이용한 상호 배제, Atomic 연산 사용, 공유 자원 사용 최소화 등의 방법을 사용합니다.
비선점형 스케줄링 알고리즘과 선점형 스케줄링 알고리즘을 설명해보세요. 비선점형(Non-preemptive)은 프로세스가 CPU를 할당받으면 작업이 끝날 때까지 CPU를 뺏기지 않는 방식(예: FCFS, SJF)입니다.
선점형(Preemptive)은 OS가 우선순위나 타임 슬라이스에 따라 실행 중인 프로세스로부터 CPU를 강제로 뺏어 다른 프로세스에 할당할 수 있는 방식(예: Round Robin, SRTF)입니다.
OS의 메모리 관리 전략 중 페이징과 세그멘테이션을 비교해서 설명해보세요. 페이징은 물리 메모리를 일정한 크기(프레임)로 나누어 할당하는 방식으로 외부 단편화는 해결하지만 내부 단편화가 발생할 수 있습니다.
세그멘테이션은 메모리를 논리적 단위(함수, 배열 등)인 세그먼트로 나누어 할당하는 방식으로 내부 단편화는 해결하지만 외부 단편화가 발생할 수 있습니다.
가상 메모리가 무엇인지와 가상 메모리가 필요한 이유를 설명해보세요. 물리 메모리 크기의 한계를 극복하기 위해 보조 기억 장치(디스크)의 일부를 메모리처럼 사용하는 기술입니다.
실제 메모리보다 큰 프로그램을 실행할 수 있게 해주고, 각 프로세스에 독립적인 가상 주소 공간을 제공하여 메모리 관리를 효율적으로 만듭니다.
스레싱(Thrashing)이 무엇인가요? 페이지 폴트가 너무 자주 발생하여 CPU가 실제 작업을 처리하는 시간보다 페이지를 교체하는 데 더 많은 시간을 소비하게 되는 현상입니다.
이로 인해 CPU 이용률이 급격히 떨어지게 됩니다.
캐시 메모리에 대해 설명해보세요. CPU와 메인 메모리(RAM) 사이의 속도 차이를 완화하기 위해 사용되는 작고 빠른 메모리입니다.
'참조 지역성' 원리를 이용하여 자주 사용될 것으로 예상되는 데이터를 미리 저장해둠으로써 시스템의 전반적인 성능을 향상시킵니다.
