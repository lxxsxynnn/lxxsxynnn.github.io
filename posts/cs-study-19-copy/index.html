[CS 스터디] 선형 자료구조

2026-06-23

1. 배열

정해진 크기만큼 데이터가 일렬로 저장되는 정적 자료구조 각 데이터를 배열의 요소라고 하며, 데이터를 가리키는 번호를 인덱스라고 함 일반적으로 배열의 인덱스는 0 base

배열에서의 시간 복잡도 접근 O(1) 검색 O(n) 삽입 O(n) 배열의 가용 공간보다 배열의 크기가 작은 경우에는 기존 배열의 마지막 인덱스 + 1로 접근해 삽입하므로 O(1)의 시간 복잡도

삭제 O(n) 삽입과 마찬가지로 마지막 요소 삭제 시 O(1)

2. 연결 리스트

크기가 정해져 있지 않은 동적 자료구조 연결 리스트가 여러 개의 노드(데이터와 다음 노드의 주소 값으로 구성)로 구성되어 있음 헤드 포인터(첫 번째 노드를 가르킴)와 테일 포인터(다음 노드 주소 값이 NULL)로 시작과 끝을 알 수 있음

노드가 메모리에 연속적으로 저장되지 않아도 필요한 연산을 수행할 수 있게 함 -> 데이터의 추가 및 삭제가 자유로움 인덱스가 없어서 특정 위치의 데이터에 접근할 때는 배열보다 시간이 오래 걸림

연결 리스트의 시간 복잡도 검색 O(n) 추가 O(n) 데이터 삽입 자체는 O(1)이지만 해당 데이터를 추가할 위치까지 이동할 때 O(n)이 소요됨

삭제 O(n) 삽입과 마찬가지로 첫번째 데이터 삭제 시 O(1), 나머지 위치는 이동 후 삭제를 해야하므로 O(n)

cf) 연결 리스트의 종류

  1. 이중 연결 리스트 앞 뒤 노드의 주소값을 모두 저장 단순 연결 리스트와 달리 양방향 탐색 가능, 노드 연결 순서와 무관하게 노드를 연속적으로 탐색해야 하는 경우 단순 연결 리스트 대비 시간 면에서 효율적 대신 구현이 더 어렵고 한 노드당 주소 값 2개를 저장해야 해서 메모리를 많이 차지함

  2. 원형 연결 리스트 마지막 노드가 NULL 값이 아니라 첫 번째 노드의 주소값을 가리킴 삽입과 삭제 연산을 효율적으로 수행 가능 새로운 노드를 맨 마지막 또는 맨 앞에 삽입하면 상수 시간이 소요됨 순환 구조라서 어느 노드든지 배열의 다른 노드에 모두 접근 가능

3. 스택

데이터를 쌓는 형태, 후입선출(LIFO) 구조 push: 데이터를 삽입하는 연산, 가장 위에 데이터가 저장됨 pop: 데이터를 삭제하는 연산, 마지막 저장 데이터를 삭제함 top이라는 변수를 이용해 마지막 저장된 데이터의 인덱스를 기억함 스택을 구현할 때는 배열과 연결 리스트를 이용

스택의 시간 복잡도 삽입(push) O(1) 삭제(pop) O(1) 확인(peek) O(1) 비어 있는지 확인(isEmpty) O(1) 가득 찼는지 확인(isFull) O(1)

작업 실행 취소, 브라우저에서 뒤로가기 등 최근에 처리한 작업들을 하나씩 꺼낼 때 사용

4. 큐

데이터가 순차적으로 들어오는 형태, 선입선출(FIFO) 구조 큐의 맨 앞을 front, 맨 뒤를 rear라고 하는데 이를 이용해 삽입과 삭제 연산의 시간 복잡도를 줄일 수 있음 인큐: 큐의 맨 뒤에 데이터 삽입 디큐: 큐의 맨 앞에서 데이터 삭제

배열과 연결 리스트를 통해 구현

큐의 시간 복잡도 rear에 삽입(enqueue) O(1) front에서 삭제(dequeue) O(1) 확인(peek) O(1) 비어 있는지 확인(isEmpty) O(1) 가득 찼는지 확인(isFull) O(1)

프로세스 할당 받기 전 대기하는 준비 큐처럼 어떤 작업을 처리할 때 작업 요청이 들어온 순서대로 처리하기 위해 주로 큐를 사용함

cf) 큐를 배열로 구현하면 rear의 인덱스를 이용해 큐가 가득 찼는지 쉽게 확인 가능 인큐 또는 디큐 수행 시 front나 rear만 수정하면 됨 다만 데이터 삽입 연산으로 큐를 꽉 채운 뒤 1개만 남기고 삭제 연산을 수행하면 rear 인덱스를 보고 큐에 데이터가 가득 찼다고 생각할 수도 있음 -> 이를 해결하려고 인큐나 디큐를 수행할 때 데이터의 위치를 옮기면 시간 복잡도가 늘어남 => 새로운 해결 방식으로 순환 큐(원형 큐) 등장 삽입 연산을 할 때 배열의 앞부분에 데이터를 삽입, 배열의 시작과 끝이 구분되지 않음

cf) 덱 양쪽 끝에서 데이터의 삽입과 삭제가 모두 가능한 자료구조 큐와 스택을 합친 형태 양방향 연결 리스트와의 차이점은?
