[CS 스터디] 비선형 자료구조

2026-06-24
  • 하나의 데이터 뒤에 N개의 데이터가 이어질 수 있는 자료구조
  • 1:N 또는 N:N 구조로 데이터가 나열되어 계층적 구조를 나타내기에 편리함
  • 데이터를 하나하나 탐색하지 않아도 원하는 데이터를 찾을 수 있음

1. 그래프(Graph)

  • 데이터를 포함하는 정점(vertex) 과 정점을 잇는 간선(edge) 으로 구성된 자료구조
  • 정점은 노드(node) 라고도 함
  • 각 용어의 영문 앞 글자를 따서 G = (V, E) 로 표현

관련 용어

용어 의미
인접 (adjacent) 두 정점이 간선으로 연결되어 있음
차수 (degree) 정점에 연결된 간선의 수
진입 차수 (in-degree) 해당 정점으로 향하는 간선의 수
진출 차수 (out-degree) 해당 정점에서 나가는 간선의 수
경로 (path) 한 정점에서 다른 정점으로 이어지는 정점들의 리스트
경로 길이 (path length) 경로를 구성하는 간선의 수
단순 경로 (simple path) 모두 다른 정점으로 구성된 경로
사이클 (cycle) 한 정점에서 시작해 같은 정점으로 돌아오는 경로 
        ┌──────────────────────┐   ← C에서 A로 (사이클 완성)
        ▼                      │
       (A) ──▶ (B) ──▶ (C) ────┘
        │
        ▼
       (D)
  • 인접: A와 B는 간선으로 직접 연결되어 있어 인접함
  • 진출 차수(A) = 2: A에서 나가는 간선 (A→B, A→D)
  • 진입 차수(A) = 1: A로 들어오는 간선 (C→A)
  • 경로: A → B → C (이때 경로 길이 = 간선 2개)
  • 사이클: A → B → C → A (출발한 정점으로 되돌아옴)

종류

  • 무방향 그래프 (undirected graph): 간선에 방향성이 없는 그래프
    • 정점이 N개일 때 최대 간선 수 = N × (N − 1) / 2
  • 방향 그래프 (directed graph): 간선에 방향성이 있는 그래프
    • 정점이 N개일 때 최대 간선 수 = N × (N − 1)
   무방향 그래프              방향 그래프
   (1)───(2)                (1)──▶(2)
    │    /                    ▲    │
    │   /                     │    ▼
   (3)─(4)                   (3)◀─(4)

cf) 그 외의 그래프

  • 부분 그래프 (subgraph): 기존 그래프에서 일부 정점·간선을 제외한 그래프
  • 가중치 그래프 (weighted graph): 간선에 비용이나 가중치가 할당된 그래프
  • 완전 그래프 (complete graph): 간선을 최대로 가진 그래프
    • 무방향이면 간선 수가 n(n−1)/2, 방향이면 n(n−1)일 때 완전 그래프
  • 유향 비순환 그래프 (DAG, Directed Acyclic Graph): 방향 그래프이면서 사이클이 없는 그래프
   가중치 그래프          완전 그래프(K4)        유향 비순환 그래프(DAG)
   (1)──5──(2)            (1)────(2)            (1)──▶(2)──▶(4)
    │       │              │ ＼ ／ │              │           ▲
    3       2              │  ╳   │              ▼           │
    │       │              │ ／ ＼ │             (3)──────────┘
   (3)──1──(4)            (3)────(4)
   간선마다 가중치 부여     모든 정점이 서로 연결   방향은 있지만 사이클 없음

경로 탐색

시작 정점이 주어졌을 때 간선을 거쳐 모든 정점을 탐색하는 방법, BFS와 DFS가 있음

        (1)
       /   \
     (2)   (3)
     / \      \
   (4) (5)    (6)

1. 너비 우선 탐색 (BFS, Breadth-First Search)

  • 시작 정점에서 가까운 정점을 먼저 탐색하는 방식
  • 먼저 발견한 정점과 인접한 정점들을 탐색하며 **큐(Queue)**에 삽입
  • 큐에 넣기 전에 방문 여부를 반드시 확인 → 안 하면 이미 방문한 정점을 다시 넣어 무한 반복
  • 비가중치 그래프에서 시작 정점부터 특정 정점까지의 최단 거리를 알 수 있음

탐색 과정 (1번에서 시작)

단계 방문 정점 큐 상태 설명
① 1 [2, 3] 1 방문, 인접 2·3을 큐에 삽입
② 2 [3, 4, 5] 2 방문, 인접 4·5를 큐에 삽입
③ 3 [4, 5, 6] 3 방문, 인접 6을 큐에 삽입
④ 4 [5, 6] 4 방문, 새 정점 없음
⑤ 5 [6] 5 방문, 새 정점 없음
⑥ 6 [] 6 방문, 큐가 비어 종료

방문 순서: 1 → 2 → 3 → 4 → 5 → 6 (레벨 순서대로 퍼져나감)

2. 깊이 우선 탐색 (DFS, Depth-First Search)

  • 시작 정점에서 탐색 가능한 최대 깊이까지 내려간 뒤,
  • 더 갈 곳이 없으면 방문한 정점들을 역순으로 거슬러 올라가며 탐색 가능한 정점이 있는지 확인
  • 갈 수 있는 정점이 있으면 거기서 다시 최대 깊이까지 탐색
  • 재귀 호출 또는 **스택(Stack)**으로 구현
  • 특정 정점에서 다른 정점까지의 경로 존재 여부를 알 수 있음
  • 재귀로 구현하면 방문할 정점을 따로 저장할 필요가 없어 BFS보다 저장 공간을 적게 사용

탐색 과정 (1번에서 시작)

단계 현재 정점 동작
① 1 1 방문 → 가장 작은 인접 정점 2로 내려감
② 2 2 방문 → 4로 내려감
③ 4 4 방문 → 더 내려갈 곳 없음, 2로 백트래킹
④ 2 남은 인접 정점 5로 내려감
⑤ 5 5 방문 → 막다른 길, 1까지 백트래킹
⑥ 3 1의 남은 인접 정점 3으로 내려감 → 6 방문

방문 순서: 1 → 2 → 4 → 5 → 3 → 6 (한 갈래를 끝까지 파고듦)

BFS는 가까운 순서대로 넓게(큐), DFS는 한 길을 끝까지 깊게(스택/재귀)
최단 거리가 필요하면 BFS, 경로 존재·완전 탐색이면 DFS가 기본 선택

2. 트리(Tree)

  • 그래프의 한 종류로 사이클이 없어 계층적 관계를 표현할 수 있음

주요 용어

용어 의미
루트 노드 부모가 없는 노드 (트리에 하나만 존재)
부모 노드 루트 노드 방향으로 연결된 노드
자식 노드 루트 노드의 반대 방향으로 연결된 노드
단말 노드 자식이 없는 노드 (잎/리프)
형제 노드 부모가 같은 노드
레벨 루트로부터의 상대적 위치 (루트의 레벨은 0)
높이 트리의 최대 레벨 + 1
차수 자식 노드의 개수

이진 트리(Binary Tree)

자식 노드가 최대 2개인 트리.

  1. 완전 이진 트리: 마지막 레벨을 제외한 모든 레벨이 채워져 있고, 마지막 레벨은 왼쪽부터 오른쪽으로 채워진 이진 트리
  2. 포화 이진 트리: 마지막 레벨까지 노드가 모두 채워진 이진 트리 (완전 이진 트리이기도 함)
  3. 이진 탐색 트리(BST): 한 노드의 왼쪽 서브트리는 그 노드보다 작은 값, 오른쪽 서브트리는 큰 값으로 구성된 트리
   완전 이진 트리        포화 이진 트리         이진 탐색 트리(BST)
        (1)                  (1)                    (8)
       /   \                /   \                  /   \
     (2)   (3)            (2)   (3)              (3)   (10)
     / \                 / \   / \              / \      \
   (4) (5)             (4)(5)(6)(7)           (1) (6)    (14)
  • 균형이 깨진 BST는 한쪽으로 치우쳐(편향 트리) 검색에 O(n) 이 걸려 BST의 장점이 사라짐
  • 그래서 항상 균형을 맞추는 균형 이진 탐색 트리 가 필요함 → 검색 O(log n) 보장
    • AVL 트리: 모든 노드에서 좌·우 서브트리의 높이 차가 1 이하 가 되도록, 삽입·삭제 때마다 회전(rotation) 으로 균형을 맞춤
    • 레드-블랙 트리: 각 노드를 빨강/검정으로 칠하고 “루트는 검정, 빨강 노드의 자식은 검정, 모든 경로의 검정 노드 수가 같다” 같은 규칙으로 균형을 유지
      AVL보다 균형은 느슨하지만 삽입·삭제가 더 빨라 실무에서 자주 쓰임

cf) 이진 탐색 트리의 데이터 추가·삭제

1. 추가 — 루트부터 값을 비교하며 내려가, 빈 자리에 삽입

   5를 삽입             (8)                  (8)
                      /   \               /   \
                   (3)   (10)   ──▶    (3)   (10)
                   / \                 / \
                 (1) (6)             (1) (6)
                                         /
     5 < 8 → 왼쪽 / 5 > 3 → 오른쪽       (5)   ← 6의 왼쪽 자식으로 삽입
   / 5 < 6 → 왼쪽 (빈자리)

2. 삭제 — 자식 노드 개수에 따라 3가지 경우로 나뉨

  • 자식이 없는 경우: 해당 노드를 그냥 삭제

          (8)              (8)
     /   \            /
   (3)   (10)  ──▶  (3)      ← 단말 노드 10 삭제

  • 자식이 1개인 경우: 삭제 후 그 자식을 부모에 바로 연결

          (8)              (8)
     /   \            /   \
   (3)  (10)   ──▶  (3)   (14)   ← 10 삭제, 자식 14를 끌어올림
            \
           (14)

  • 자식이 2개인 경우: 오른쪽 서브트리의 최솟값(또는 왼쪽 서브트리의 최댓값)을 찾아 삭제할 노드 자리에 올림 (BST 정렬 순서 유지)

          (8)                     (10)
     /   \                   /    \
   (3)   (14)    ──▶       (3)    (14)   ← 8 삭제,
         /  \                     /      오른쪽 서브트리 최솟값 10을 올림
      (10) (16)                (16)

3. 우선순위 큐(Priority Queue)

  • 우선순위가 높은 데이터가 먼저 나오는 자료구조
  • 큐와 동일하게 삽입·삭제 연산을 지원하되, 삭제 시 우선순위가 가장 높은 데이터가 나옴
  • 배열, 연결 리스트, 힙(heap) 등으로 구현

구현 방식별 시간 복잡도

구현 방법 삽입 삭제
배열 (정렬 안 됨) O(1) O(n)
연결 리스트 (정렬 X) O(1) O(n)
배열 (정렬됨) O(n) O(1)
연결 리스트 (정렬됨) O(n) O(1)
힙 O(log n) O(log n)

삽입·삭제가 모두 O(log n)으로 균형이 좋아, 우선순위 큐는 보통 힙으로 구현합니다.

4. 힙(Heap)

  • 완전 이진 트리이면서, 최댓값 또는 최솟값을 빠르게 찾을 수 있는 자료구조
  • 우선순위 큐 구현에 자주 사용됨
  • 최대 힙: 부모 노드 값 ≥ 자식 노드 값 → 루트가 최댓값
  • 최소 힙: 부모 노드 값 ≤ 자식 노드 값 → 루트가 최솟값

1. 삽입 — 맨 끝에 넣고, 부모와 비교하며 위로 올림 (상향식, up-heap)

   최대 힙에 14 삽입
        (10)                (10)                  (14)
       /    \      넣기      /    \     비교·교환    /    \
     (8)    (5)   ──▶     (8)    (5)    ──▶       (10)   (5)
     /                    / \                     / \
   (3)                  (3)(14)                 (3) (8)
                         14 > 8 → 교환            14 > 10 → 교환 (루트 도달)

2. 삭제 — 루트(최우선 데이터)를 꺼내고, 마지막 노드를 루트로 옮긴 뒤 아래로 내림 (하향식, down-heap)

   최대 힙에서 삭제(루트 14 제거)
        (14)                (8)                   (10)
       /    \    마지막을    /    \    비교·교환      /    \
     (10)   (5)  루트로    (10)   (5)   ──▶       (8)    (5)
     /            ──▶
   (8)
                8을 루트로       10 > 8 → 교환 → 재정렬 완료

5. 해시 테이블(Hash Table)

  • 하나의 **키(key)**에 대해 하나의 **값(value)**을 저장하는 자료구조
  • 해시 함수에 키를 넣으면 해시(= 값이 저장된 테이블의 인덱스)를 얻음
  • 매칭되는 결과 값에 한 번에 접근할 수 있어 평균 **O(1)**의 시간 복잡도
   key "apple" ──▶ [해시 함수] ──▶ 인덱스 2
        ┌─────┬─────┬───────┬─────┐
        │  0  │  1  │   2   │  3  │
        └─────┴─────┴───────┴─────┘
                        ▲
                   "apple" 값 저장
  • 서로 다른 키가 같은 해시를 갖는 **해시 충돌(collision)**이 발생할 수 있음

해결 방법

  1. 체이닝(Chaining): 충돌 시 같은 해시의 값들을 연결 리스트로 연결 해 저장

    • 저장 공간 제약이 적지만, 한 해시에 노드가 몰리면 그 부분 탐색이 느려짐
       인덱스 2 ──▶ ["apple"] ──▶ ["grape"] ──▶ NULL

  2. 개방 주소법(Open Addressing): 충돌 시 다른 빈 공간 에 값을 저장

    • 선형 조사법: 충돌 지점에서 한 칸씩(+1, +2, …) 옮겨가며 빈자리 탐색
    • 이차 조사법: 충돌 지점에서 제곱(+1², +2², …) 간격으로 탐색해 한곳에 몰리는 현상 완화
    • 이중 해싱: 또 다른 해시 함수로 이동 폭을 정해 분포를 더 고르게 함
