[CS 스터디] 네트워크 계층

2026-06-05

1. OSI 7계층 (OSI 7 Layer Model)

네트워크 통신이 이뤄지는 과정을 7단계로 나눈 네트워크 표준 모델

  • 데이터 흐름:
    • 송신부: 높은 계층(L7)에서 낮은 계층(L1)으로 데이터를 전달함
    • 수신부: 가장 낮은 계층(L1)에 도달한 데이터를 다시 높은 계층(L7)으로 전달함
  • 캡슐화 및 역캡슐화:
    • 각 계층은 독립적이며, 데이터를 송신할 때 각 계층에서 필요한 정보를 추가하여 데이터를 가공함
    • 제어 정보를 담은 헤더(Header) 나 트레일러(Trailer) 를 붙이는 과정을 캡슐화(Encapsulation) 라고 함
    • 수신 측의 같은 계층에서는 해당 정보를 확인하고 제거하며 상위 계층으로 전달하는 역캡슐화(Decapsulation) 를 수행함
    • 이를 통해 데이터 호환성을 높이고 오류의 영향을 최소화할 수 있음

계층 구조 및 캡슐화 흐름 도식화 

      [ 송신부 (Sender) ]                          [ 수신부 (Receiver) ]
    L7. 응용       [ Data ]             │        L7. 응용       [ Data ]
                     ↓                 │                         ↑
    L6. 표현    [H6][ Data ]            │        L6. 표현    [H6][ Data ]
                     ↓                 │        (역캡슐화)          ↑
    L5. 세션    [H5][ Data ]          (수신)      L5. 세션    [H5][ Data ]
                     ↓                 │                          ↑
    L4. 전송    [H4][ Data ]            │        L4. 전송    [H4][ Data ]
      (캡슐화)       ↓                   │                         ↑
    L3. 네트워크 [H3][ Data ]            │        L3. 네트워크  [H3][ Data ]
                     ↓                 │                         ↑
    L2. 데이터  [H2][ Data ][T2]         │        L2. 데이터  [H2][ Data ][T2]
                     ↓                 │                         ↑
    L1. 물리    [ 010101... ]  ───── (전송) ────>  L1. 물리    [ 010101... ]
  • 계층별 역할:
    • 7계층 (응용): HTTP, FTP 등의 프로토콜을 응용 프로그램의 UI를 통해 제공
    • 6계층 (표현): 데이터를 표준화된 형식으로 변경 및 암호화, 압축 수행
    • 5계층 (세션): 세션의 유지 및 해제 등 응용 프로그램 간 통신 제어와 동기화
    • 4계층 (전송): 신뢰성 있는 데이터를 전달하기 위한 계층. TCP/UDP 전송 방식과 포트 번호 결정
    • 3계층 (네트워크): 데이터를 목적지까지 보내기 위한 최적 경로를 선택하는 라우팅 수행 (IP, 라우터)
    • 2계층 (데이터 링크): 데이터 흐름 관리 및 오류 검출, 복구 수행 (MAC, 스위치)
    • 1계층 (물리): 데이터를 비트 단위(0과 1)로 변환하여 전송하거나 전기 신호를 데이터로 복원 (허브, 리피터)

2. TCP/IP 4계층 (TCP/IP 4 Layer Model)

인터넷에서 데이터를 주고받기 위한 실제 네트워크 프로토콜 스택

  • 특징:
    • TCP(패킷 전송 보장)와 IP(빠른 경로 전송)를 중심으로 OSI 7계층을 단순화한 모델
    • 각 계층마다 데이터를 부르는 전송 단위(PDU)가 다름

계층별 데이터 흐름 및 전송 단위 

    [ 계층 (Layer) ]           [ 전송 단위 (PDU) ]        [ 주요 프로토콜 ]
   4. 응용 (Application)   ──       Data        ──    HTTP, HTTPS, DNS, FTP
   3. 전송 (Transport)     ──     Segment       ──    TCP, UDP
   2. 인터넷 (Internet)     ──  Packet/Datagram  ──    IP, ICMP, ARP
   1. 네트워크 인터페이스      ──      Frame        ──    Ethernet, Wi-Fi, MAC
  • 계층별 상세:
    • 4. 응용 계층: 사용자와 소프트웨어를 연결해 주는 계층으로 다양한 서비스 제공
    • 3. 전송 계층: 데이터의 신뢰성을 보장하며, 포트 번호를 통해 적절한 응용 프로그램에 데이터 전달
    • 2. 인터넷 계층: 데이터를 최종 목적지까지 도달하게 함
      전송 계층 데이터에 헤더를 붙여 패킷으로 캡슐화
    • 1. 네트워크 인터페이스 계층: 데이터를 전기 신호로 변환하고 MAC 주소를 사용하여 물리적 기기에 데이터 전달

3. 주요 용어 정리

  • 패킷 (Packet): 네트워크에서 분할하여 전송하는 데이터의 기본 단위
    송수신 주소 정보 포함
  • MAC 주소: 하드웨어 고유의 48비트 주소
    네트워크 인터페이스 계층(L1/L2)에서 사용
  • IP 주소: 네트워크 장치를 식별하기 위한 논리적 주소
    네트워크부(네트워크 구분)와 호스트부(기기 구분)로 나뉨
  • 서브넷 마스크 (Subnet Mask): IP 주소에서 네트워크 영역과 호스트 영역을 구분하기 위해 사용하는 비트 마스크
