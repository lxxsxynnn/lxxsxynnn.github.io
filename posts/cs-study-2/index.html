[CS 스터디] 프로세스와 스레드

2026-06-02

1. 프로세스와 스레드

프로세스(Process): 컴퓨터에서 실행 중인 프로그램. OS로부터 독립된 메모리 영역(코드, 데이터, 스택, 힙)을 할당받음

  • 프로그램: 특정 작업을 수행하기 위한 명령어 집합 (정적 상태)
  • 메모리 영역 구조:
    [ 높은 주소 ]
        ↓
  ┌────────────┐
  │    스택     │ ← 지역 변수, 함수의 매개변수, 반환 주소 (LIFO)
  ├────────────┤
  │      ↓     │ ← 빈 공간 (Stack 영역)
  ├────────────┤
  │      ↑     │ ← 빈 공간 (Heap 영역)
  ├────────────┤
  │      힙     │ ← 동적 메모리 할당 (malloc, new 등 / FIFO)
  ├────────────┤
  │    데이터    │ ← 전역 변수, 정적 변수 (BSS/Data 영역으로 세분화)
  ├────────────┤
  │    코드     │ ← 실행할 기계어 명령어 (Text 영역)
  └────────────┘
        ↑
    [ 낮은 주소 ]
  • 스택 오버플로우: 스택 영역이 힙 영역을 침범
  • 힙 오버플로우: 힙 영역이 스택 영역을 침범

스레드(Thread): 프로세스 내에서 실제로 작업을 수행하는 실행 흐름의 단위

  • 특징: 프로세스 내 힙, 데이터, 코드 영역 공유. 개별 스택 영역 할당
  • 스레드 모델:
    1. 다대일 모델: 사용자 스레드 n개 - 커널 스레드 1개. 병렬성 이용 불가
    2. 일대일 모델: 사용자 스레드 1개 - 커널 스레드 1개. 성능 저하 가능성 존재
    3. 다대다 모델: n개 사용자 스레드 - m개 커널 스레드 매핑 (m <= n)

2. PCB (Process Control Block)

PCB: OS가 프로세스를 제어하기 위해 정보를 저장하는 블록

  • 저장 정보: 포인터, 프로세스 상태, PID(고유 ID), PC(다음 실행 명령어 주소), 레지스터, 메모리 제한, 열린 파일 목록 등

3. 프로세스의 생성과 종료

  • 생성: 부모 프로세스에서 fork() 호출. 부모는 자식의 PID를, 자식은 0을 반환
  • 종료 조건:
    • exit() 호출을 통한 정상 종료
    • 제한 시간 초과, 파일 검색 실패, 메모리 부족 등 오류 발생
    • 부모 프로세스가 자식을 강제 종료 (자원 초과 사용 등)

4. 프로세스 상태와 변화 

       [생성(New)]
            ↓ 승인(Admit)
       [준비(Ready)] <─────── 인터럽트/타임아웃 ────────┐
            ↓ 디스패치(Dispatch)                     │
       [실행(Running)] ────────────────────> [종료(Terminated)]
            │                                    완료
            └──────── 입출력/이벤트 대기 ────────> [대기(Waiting)]
                                                   │
            <──────── 입출력/이벤트 완료 ──────────────┘
  • 상태 변화:
    • 생성 → 준비: OS 승인을 받아 준비 큐(Ready Queue)에 추가됨
    • 준비 → 실행: 우선순위가 높은 프로세스가 디스패치되어 CPU 점유
    • 실행 → 준비: 타임 슬라이스 만료(Timeout)로 인한 상태 복귀
    • 실행 → 대기: 입출력 또는 특정 이벤트 대기 발생
    • 대기 → 준비: 대기 중인 이벤트가 완료되어 다시 CPU 할당 대기

5. 멀티 프로세스와 멀티 스레드

  • 동시성(Concurrency): 싱글 코어에서 작업을 번갈아 처리 (논리적 동시)
  • 병렬성(Parallelism): 멀티 코어에서 작업을 동시 처리 (물리적 동시)
구분 멀티 프로세스 멀티 스레드
안정성 높음 (한 프로세스 고장이 파급되지 않음) 낮음 (한 스레드 오류가 프로세스 전체 영향)
자원 효율 낮음 (개별 메모리 할당으로 비용 높음) 높음 (메모리 공유로 자원 절약)
콘텍스트 스위칭 무거움 (메모리 영역 교체 오버헤드 큼) 가벼움 (공유 자원이 많아 전환 빠름)
통신(IPC) 복잡함 (별도 통신 메커니즘 필요) 단순함 (공유 메모리 직접 접근)

6. 문맥 교환 (Context Switching)

문맥 교환: 인터럽트 발생 시 현재 프로세스 상태를 PCB에 저장하고 새 프로세스의 PCB 정보를 로드하는 과정

  • 저장 위치: PCB 내 프로그램 카운터(PC)와 스택 포인터(SP)
프로세스 P0            운영체제(커널)            프로세스 P1
    │                     │                     │
   실행중                   │                     │
    │   인터럽트/시스템 콜     │                     │
    ├────────────────────>│                     │
    │                     │ P0 상태를 PCB0에 저장   │
   대기중                   │                     │
    │                     │ PCB1에서 P1 상태 로드  │
    │                     │                     │
    │<────────────────────┤                     │
    │                     │                   실행중
    │                     │                     │

7. 프로세스 동기화

  • 임계 영역(Critical Section): 공유 자원에 접근하여 실행 순서에 따라 결과가 변하는 코드 영역
  • 동기화 조건: 상호 배제(Mutual Exclusion), 진행(Progress), 한정된 대기(Bounded Waiting)
  • 기법:
    • 뮤텍스(Mutex): 락(Lock) 기반 제어. 대기 시 스핀락(Spin-lock) 사용 가능
    • 세마포어(Semaphore): 카운트 기반 신호(Signaling) 제어
  • 주요 개념:
    • 동기/비동기: 작업 순서 보장 여부
    • 블로킹/넌블로킹: 작업 수행 시 대기 여부

8. 교착 상태 (Deadlock)

정의: 프로세스들이 서로가 점유한 자원을 무한히 기다리는 상태

  • 필요 충분 조건: 상호 배제, 점유와 대기, 비선점, 환형 대기
  • 해결 방안(부정):
    • 상호 배제 부정: 자원 공유 허용
    • 점유와 대기 부정: 실행 전 필요 자원 일괄 할당
    • 비선점 부정: 자원 요청 시 점유 자원 반납
    • 환형 대기 부정: 자원 번호 순서대로 요청

9. 스레드 안전 (Thread Safe)

정의: 멀티 스레드 환경에서 공유 자원 접근 시 데이터 일관성이 보장되는 상태

  • 달성 조건: 상호 배제, 원자 연산(Atomic Operation), 재진입성(Reentrancy), 스레드 지역 저장소(TLS)

10. IPC (Inter-Process Communication)

  1. 공유 메모리: 프로세스 간 공유 메모리 구간 설정 (가장 빠름)
  2. 소켓: 네트워크 기반 통신 (원격 시스템 가능)
  3. 세마포어: 동기화를 통한 자원 관리
  4. 파이프: FIFO 단방향 통신 (양방향 시 파이프 2개 필요)
  5. 메시지 큐: 큐 자료구조 기반 메시지 전달

11. 좀비 및 고아 프로세스

  • 좀비 프로세스: 자식 종료 후 부모가 상태를 회수하지 않아 시스템에 남은 상태
  • 고아 프로세스: 부모가 먼저 종료된 상태(PID를 1로 바꿔주면 자식이 좀비 프로세스가 되는 걸 막을 수 있음)
