이번 주 키워드

변수, 설계

변수란?

자바에서 변수는 타입 + 이름으로 구성
이 변수를 통해 쓰기(= 연산자), 읽기(변수명) 등의 작업을 수행 가능

변수를 사용하는 이유?
→ 컴파일러가 코드를 분석하고 오류를 사전에 잡을 수 있도록 하기 위해서

컴파일러는 다음과 같은 작업을 수행:

  • 문법 체크
  • 최적화
  • 번역 (.java → .class)

📌 즉, 코드가 완벽하지 않아도 컴파일러가 이를 어느 정도 보완해줌


변수의 종류

자바의 변수는 크게 두 가지:

  • 기본형(primitive): 하나의 요소
  • 참조형(reference): 여러 개의 요소(집합)

참조형은 0~n개의 요소를 담는 집합이며,
이 집합을 만들기 위해서는 기준을 가지고 그룹화를 수행
➡️ 이것이 바로 설계


설계란?

관련 있는 것끼리 하나의 집합으로 묶고 집합들 간의 관계를 정의

설계를 하면 좋은 점:

  • 복잡도가 낮아지고
  • 관리가 쉬워지며
  • 더 복잡한 프로그램을 구현할 수 있게 됨

연산

연산이란 데이터를 계산하는 것

  • 같은 타입끼리만 연산 가능
  • 반드시 결과를 반환해야 함

연산자에서 중요한 3요소:

  1. 우선순위
  2. 결합법칙
  3. 산술 변환
    • 둘 중 큰 타입으로 자동 변환
    • int보다 작은 타입끼리 연산 시 → int로 변환 후 연산

연습문제 정리

  1. long형 변수에 정수값 저장 시 L을 붙이지 않으면 기본적으로 int로 인식
    → int 범위 초과 시 컴파일 에러 발생

  2. '+' 연산자는

    • 둘 다 숫자 → 덧셈
    • 하나라도 String → 문자열 이어붙이기
    • char + char / char + 숫자 → int 반환

    단, ++, -- 같은 증감 연산자는 형변환 없이 char로 반환

  3. 자바는 C처럼 0 = false, 1 = true를 인정하지 않음
    → boolean에는 반드시 true / false 를 사용해야 함

  4. char는 요소형 → 반드시 문자를 지정해야 함
    → '' (빈 문자 리터럴) 사용 불가


자바에서의 var

Java 10부터 도입된 문법으로, 로컬 변수의 타입을 컴파일러가 자동 추론
(컴파일 타임에 정적으로 결정, 런타임에 동적으로 바뀌지 않음)
적절한 상황에 사용하면 코드의 가독성과 간결성이 좋아짐


1. 타입이 너무 길어서 코드가 지저분할 때 

// before
Map<String, List<Integer>> map = new HashMap<>();

// after
var map = new HashMap<String, List<Integer>>();

2. 우측 표현식으로 타입이 분명히 드러날 때 

var name = "Soyeon";      // String
var count = 123;          // int
var list = List.of(1, 2); // List<Integer>

3. for-each 루프에서 반복 변수 

for (var item : items) {
    System.out.println(item);
}

📌 주의:

  • var는 명시적인 타입 지정이 아닌 추론이기 때문에, 복잡한 표현식에서는 오히려 가독성을 해칠 수 있음
  • 클래스 필드, 메서드 매개변수, 리턴 타입에는 사용할 수 없음 (오직 로컬 변수에만 사용 가능)

→ 사람이 읽을 수 있을 만큼만 추론을 맡기고 그렇지 않으면 명시적으로 타입을 작성하자!
